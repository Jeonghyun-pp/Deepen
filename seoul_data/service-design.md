# 안전귀가Navi — 서비스 전체 설계

> 2026 서울시 빅데이터 활용 경진대회 · 창업 부문 (제품·서비스 개발)

---

## 1. 서비스 한 줄 정의

**"주소를 입력하면 안전 점수를 알려주고, 집까지 가장 안전한 길을 안내하는 AI 서비스"**

---

## 2. 사용자 시나리오

### 시나리오 A: 이사 전 — "이 집, 안전할까?"

```
사용자(27세 여성, 1인가구 이사 예정)

1. 직방에서 관악구 봉천동 원룸 3개 후보를 발견
2. 안전귀가Navi에 주소 3개를 입력
3. 각 주소의 안전 스코어(100점 만점)를 비교
4. AI가 생성한 리포트 확인:
   "봉천동 OO길 12는 안전 점수 68점입니다.
    ✅ CCTV 12대가 반경 500m 내 설치 (평균 대비 양호)
    ⚠️ 가로등 사각지대 3곳 (특히 골목 진입로)
    ⚠️ 가장 가까운 편의점까지 도보 8분
    ✅ 파출소 도보 5분 거리
    📊 해당 동 5대 범죄: 전년 대비 12% 감소 추세
    
    💡 비슷한 월세(45~55만원)에서 안전 점수가 더 높은 근처 동네:
    → 신림동 OO로 (78점), 낙성대 OO길 (74점)"
5. 비교 리포트를 PDF로 저장해서 부모님께 공유
```

### 시나리오 B: 매일 밤 — "오늘 어떤 길로 갈까?"

```
사용자(같은 여성, 봉천동 이사 완료)

1. 밤 11시, 강남역에서 퇴근
2. 안전귀가Navi 실행 → "귀가 시작" 터치
3. AI가 3개 경로 제안:
   🟢 추천 경로 (안전 점수 87): 도보 18분
      → CCTV 커버 구간 82%, 편의점 2곳 경유, 가로등 밝은 길
   🟡 일반 경로 (안전 점수 64): 도보 14분
      → 최단이지만 CCTV 사각지대 200m 구간 존재
   🔴 비추천 경로 (안전 점수 41): 도보 12분
      → 유흥업소 밀집, 가로등 없는 골목 포함

4. 추천 경로 선택 → 턴바이턴 안내 시작
5. 경로 중 "50m 앞 편의점 CU — 필요시 대피 가능" 알림
6. 집 도착 → "안전 귀가 완료 ✓" → 설정한 비상연락처에 자동 알림
```

### 시나리오 C: 위급 상황 — "지금 불안해"

```
1. 귀가 중 뒤에서 발소리가 계속 따라옴
2. 화면의 "긴급" 버튼 터치
3. 즉시:
   → 현재 위치 + 실시간 경로를 비상연락처에 전송
   → 가장 가까운 편의점/경찰서 위치 + 도보 경로 표시
   → 112 원터치 연결 버튼 활성화
   → (선택) 주변 CCTV 관제센터에 모니터링 요청 연동
```

## 3. 핵심 기능 구조

```
안전귀가Navi
│
├── 🏠 안전 스코어 리포트 (이사 전)
│   ├── 주소 검색 → 안전 점수 (0~100)
│   ├── 세부 항목별 점수 (CCTV, 가로등, 범죄, 인프라, 환경)
│   ├── AI 자연어 리포트 생성
│   ├── 주소 비교 (최대 3곳)
│   ├── 시간대별 안전도 변화 그래프 (낮 vs 밤)
│   └── PDF 리포트 다운로드 & 공유
│
├── 🚶 안전 귀가 내비 (매일 밤)
│   ├── 안전 경로 추천 (최단 vs 최안전 트레이드오프)
│   ├── 턴바이턴 내비게이션
│   ├── 경로 상 안전 시설 표시 (편의점, CCTV, 가로등)
│   ├── 실시간 일몰/기상 반영
│   ├── 안전 귀가 완료 알림 (비상연락처)
│   └── 귀가 타이머 (예상 시간 초과 시 알림)
│
├── 🗺️ 안전 지도 (탐색)
│   ├── 서울시 전체 안전 히트맵
│   ├── CCTV/가로등/편의점 레이어 토글
│   ├── 자치구별 범죄 통계 시각화
│   └── 시간대별 안전도 변화 (슬라이더)
│
├── 🚨 긴급 모드
│   ├── 원터치 112 신고
│   ├── 비상연락처 위치 공유
│   ├── 가장 가까운 안전 장소(편의점/파출소) 안내
│   └── 안심이 앱 CCTV 관제센터 연동 (향후)
│
└── 👤 마이페이지
    ├── 자주 가는 경로 저장
    ├── 출퇴근 시간 설정 → 개인화 위험도 반영
    ├── 비상연락처 관리
    └── 귀가 이력 & 통계
```

## 4. 안전 점수 산출 모델

### 4.1 입력 변수 (Feature)

| 카테고리 | 변수 | 데이터 소스 | 가중치(초기) |
|---|---|---|---|
| **치안** | 반경 500m 5대 범죄 발생 건수 | 서울시 5대 범죄 발생현황 통계 | 25% |
| **감시** | 반경 500m CCTV 대수 / 커버리지 | 서울시 CCTV 설치현황 + 안심이 CCTV 연계현황 | 20% |
| **조명** | 반경 500m 가로등 밀도 + 사각지대 수 | 서울시 가로등 위치 정보 + 점소등 시간 | 15% |
| **대피** | 가장 가까운 편의점/파출소까지 거리 | 공공데이터포털 편의점/경찰서 인허가 데이터 | 15% |
| **환경** | 유흥업소 밀집도 | 서울시 유흥업소 인허가 정보 | 10% |
| **도로** | 골목길 폭, 보행로 유무 | 서울시 보행자 도로 데이터 | 10% |
| **시간** | 현재 시간 vs 일몰 시간, 유동인구 | 기상청 일몰 API + 서울시 생활인구 데이터 | 5% |

### 4.2 점수 산출 파이프라인

```
[주소 입력]
    │
    ▼
[지오코딩] → 위도/경도 변환 (카카오맵 API)
    │
    ▼
[반경 검색] → 500m 버퍼 내 공공데이터 수집
    │
    ├── CCTV 위치 → 밀도 계산 + 사각지대 식별
    ├── 가로등 위치 → 조도 커버리지 추정
    ├── 범죄 이력 → 연도별 추세 + 유형별 분류
    ├── 편의점/파출소 → 최단 거리 계산
    ├── 유흥업소 → 밀집도 계산
    └── 보행 도로 → 폭원, 보도 유무
    │
    ▼
[ML 스코어링]
    │
    ├── 초기: 가중합 모델 (위 가중치 기반)
    │   Score = Σ(변수_i × 가중치_i) → 0~100 정규화
    │
    └── 고도화: XGBoost / LightGBM
        - 학습 데이터: 범죄 발생 지점 주변 특성 → 위험도 레이블
        - 시간대별 모델 분리 (주간/야간/심야)
        - Feature importance → 가중치 자동 학습
    │
    ▼
[생성형 AI 리포트]
    │
    ├── 입력: 점수 + 항목별 수치 + 주변 맥락
    ├── 프롬프트 엔지니어링으로 일관된 형식 생성
    └── 출력: 자연어 안전 리포트 (강점/약점/대안 추천)
```

### 4.3 시간대별 동적 점수

같은 주소라도 시간대에 따라 안전 점수가 달라짐:

| 시간대 | 변동 요소 | 점수 영향 |
|---|---|---|
| 06~18시 (주간) | 유동인구 높음, 자연광 충분 | 기본 점수 유지 |
| 18~21시 (초저녁) | 일몰 후 가로등 점등, 퇴근 유동인구 | 가로등 가중치 ↑ |
| 21~01시 (야간) | 유동인구 감소, 유흥가 활성화 | 범죄·유흥업소 가중치 ↑↑ |
| 01~06시 (심야) | 유동인구 최저, 편의점 일부 폐점 | 전 항목 가중치 재조정, 편의점 영업 여부 반영 |

## 5. 안전 경로 알고리즘

### 5.1 기본 원리

일반 내비게이션: **최단 거리/시간** 최적화
안전귀가Navi: **안전도 가중 최단 경로** 최적화

```
경로 비용 = α × 거리(m) + β × 위험도(0~1)

α: 거리 패널티 (사용자 설정 가능)
β: 안전 패널티 (기본값 높게 설정)

→ 위험한 구간은 "거리가 멀어도" 우회
→ CCTV 있는 길, 가로등 밝은 길, 편의점 경유 경로 선호
```

### 5.2 도로 구간별 안전 점수

```
[도로 네트워크 그래프]
    │
    각 엣지(도로 구간)에 안전 점수 부여:
    │
    ├── CCTV 커버 여부 (커버 → +점)
    ├── 가로등 존재 여부 + 밀도 (밝은 길 → +점)
    ├── 편의점/상점 인접 여부 (유동인구 프록시 → +점)
    ├── 유흥업소 인접 여부 (→ -점)
    ├── 도로 폭 (넓은 길 → +점)
    └── 시간대별 유동인구 (많으면 → +점)
    │
    ▼
    Dijkstra / A* 알고리즘 (안전 가중치 적용)
    │
    ▼
    [상위 3개 경로 제안]
    ├── 🟢 추천 (가장 안전)
    ├── 🟡 균형 (안전+거리 절충)
    └── 🔴 최단 (거리 최소, 안전 보장 안 됨)
```

### 5.3 실시간 반영 요소

| 요소 | 소스 | 반영 방식 |
|---|---|---|
| 일몰 시간 | 기상청 API (일출/일몰) | 일몰 후 → 가로등 없는 구간 패널티 증가 |
| 날씨 | 기상청 단기예보 API | 비/안개 → 가시성 낮은 구간 패널티 증가 |
| 현재 시각 | 시스템 시간 | 심야 시간대 → 전체 위험도 가중치 상향 |

### 6.2 프로토타입 기술 선택

| 레이어 | 기술 | 이유 |
|---|---|---|
| **프론트엔드** | React + Vite | 빠른 프로토타이핑, 지도 SDK 호환 |
| **지도** | 카카오맵 JavaScript SDK | 한국 주소 체계 완벽 지원, 무료 |
| **백엔드** | FastAPI (Python) | ML 모델 통합 용이, 비동기 처리 |
| **DB** | PostgreSQL + PostGIS | 공간 쿼리 (반경 검색, 경로 계산) |
| **ML** | scikit-learn → XGBoost | 초기 가중합 → 점진적 고도화 |
| **경로 엔진** | OSMnx + NetworkX | OpenStreetMap 기반 보행 경로 그래프 |
| **LLM** | Claude API | 자연어 리포트 생성 |
| **배포** | Docker + (Render or Railway) | 경진대회용 빠른 배포 |

## 10. 대회 심사 기준 매핑

| 심사 항목 (배점) | 안전귀가Navi 대응 |
|---|---|
| **공공데이터 활용 (25점)** | 열린데이터광장 데이터 8종+ 결합. CCTV·가로등·범죄·유흥업소·생활인구 등 다분야 데이터를 하나의 안전 점수로 융합 — 단순 조회가 아닌 "가치 창출형" 활용 |
| **AI 혁신성 (20점)** | ① ML 기반 안전 스코어링 (가중치 자동 학습) ② 생성형 AI 자연어 리포트 ③ AI 기반 안전 경로 최적화 — 3가지 AI 기술 동시 활용 |
| **독창성 (15점)** | 국내외 "주거 안전 스코어 + 안전 귀가 경로"를 통합한 서비스 전무. 작년 수상작(복덕방AI)과 관점이 완전히 다름 (가격/면적 vs 안전) |
| **완성도 (15점)** | 실제 작동하는 웹 프로토타입. 주소 입력→점수→리포트→경로 추천까지 엔드투엔드 구현 |
| **발전 가능성 (20점)** | B2C 구독 + B2B API(부동산 앱 연동) + 기업 SaaS. 글로벌 개인안전 앱 시장 $18억→$77억(CAGR 15.6%) |
| **ESG 혁신 (5점)** | 여성·사회적 약자 안전 (Social). 서울시 기존 정책(SS존, 안심귀가) 효과 극대화 |
