> 작성일: 2026-04-10
> 목적: "안전귀가Navi" 서비스의 주거 안전 스코어링 기능 설계를 위한 근거 자료

---

## 1. 핵심 발견: 다방이 이미 하고 있다 (그러나 부족하다)

### 다방 "안전 정보 서비스" (2019.12 출시)

| 항목 | 내용 |
|---|---|
| 제공 데이터 | CCTV, 경찰서, 파출소, 치안센터, 여성안심지킴이집 **위치** |
| 시각화 | 안전시설 많은 지역 = 녹색, 적은 지역 = 다른 색 |
| 매물 연동 | 매물 상세에서 가장 가까운 치안시설까지 **거리** 표시 |
| 조회 인기 지역 | 봉천동, 신림동 (여성 1인가구 밀집 지역) |

**다방이 하는 것:**
- 치안 시설의 **위치 표시** (지도 핀)
- 매물~치안시설 **거리 계산**

**다방이 안 하는 것:**
- ❌ 실제 **범죄 발생 데이터** (5대 범죄 건수, 유형, 추세)
- ❌ 종합 **안전 점수** (스코어링)
- ❌ **가로등** 밀도/사각지대
- ❌ **유흥업소** 밀집도
- ❌ **시간대별** 안전도 변화
- ❌ **AI 분석 리포트** (자연어 해석)
- ❌ 주소 간 **비교** 기능
- ❌ **귀가 경로** 안전 분석

> **결론: 다방은 "안전 시설이 어디 있는지"를 보여줄 뿐, "여기가 얼마나 안전한지"를 판단해주지 않는다. 안전귀가Navi의 차별점은 "시설 위치 → 종합 판단"으로의 도약이다.**

### 공공 서비스 (부동산 앱과 분리)

| 서비스 | 내용 | 한계 |
|---|---|---|
| **행정안전부 생활안전지도** (safemap.go.kr) | 6대 분야 134종 지도, 행정동별 범죄 1~5등급 | 부동산 앱과 **미연동**, 별도 접속 필요 |
| **스마트치안 빅데이터 플랫폼** (bigdata-policing.kr) | 25개구 28종 치안 데이터 | 전문가용, 일반 시민 접근성 낮음 |

## 3. 해외 주거안전 비교 서비스 상세 분석

### 3.1 서비스별 "안전 정의" 비교

#### AreaVibes — 가장 종합적인 동네 비교 플랫폼

| 항목 | 내용 |
|---|---|
| **총점** | Livability Score (0~100) |
| **카테고리** | 9개: 범죄, 편의시설, 생활비, 통근, 고용, 건강, 학교, 주거, 사용자평가 |
| **비교 기능** | ✅ 도시/동네/주소 간 비교, **사용자가 가중치 커스터마이즈 가능** |
| **가격** | 무료 |
| **특징** | "나한테 중요한 게 뭔지"에 따라 점수가 달라지는 개인화 |

AreaVibes의 핵심 교훈: **사용자가 가중치를 직접 조절할 수 있다**
- 예: "나는 학군은 상관없고 치안이 가장 중요해" → 치안 가중치 ↑ → 점수 재계산
- 안전귀가Navi에 적용: 1인가구 여성 → 야간 안전/CCTV 가중치 자동 상향

#### SaferHood AI — AI 기반 주소별 리포트

| 항목 | 내용 |
|---|---|
| **분석 변수** | 15개+ |
| **범죄 세부** | 20종+ (폭력, 절도, 성범죄, 차량, 방화 등 유형별) |
| **비범죄 변수** | 학교 안전/괴롭힘, 병원 품질, 대기질, 화학시설, 갱 활동, 홍수, 화재 |
| **AI 역할** | 가중치 자동 학습, "실제 영향도"에 따른 스코어링 |
| **비교 기능** | ❌ (주소별 개별 리포트) |
| **가격** | $9.99~$34.99 (개인), $29.99/월 (중개인) |

SaferHood AI의 핵심 교훈: **"안전"을 범죄만으로 정의하지 않는다**
- 대기질, 화학시설, 홍수 위험까지 포함 → "이 동네에서 사는 게 안전한가"의 범위를 넓힘
- 안전귀가Navi에 적용: 대기과학 전공 살려서 기상/환경 변수도 포함 가능

#### CrimeGrade.org — ML 기반 범죄 등급

| 항목 | 내용 |
|---|---|
| **변수** | 180개+ |
| **등급** | A+ ~ F (센서스 블록 단위) |
| **데이터** | FBI UCR + 18,000개 지역 경찰 기관 |
| **ML 활용** | 데이터 공백 보정, 리포팅 오류 교정, 미래 범죄 예측 |
| **비교 기능** | ❌ |
| **가격** | 무료 (광고 기반) |

CrimeGrade의 핵심 교훈: **ML로 데이터 공백을 메운다**
- 모든 동네에 범죄 데이터가 균일하게 있지 않음 → ML이 주변 데이터로 추정
- 안전귀가Navi에 적용: 자치구 단위 → 동 단위로 세분화할 때 ML 추정 필요

#### Niche.com — 동네 등급 + 리뷰

| 항목 | 내용 |
|---|---|
| **등급** | A+ ~ F |
| **안전 점수 구성** | FBI UCR 범죄율 + 주민 리뷰 (정량+정성 혼합) |
| **방법론** | z-score 정규분포 기반 상대 등급 |
| **비교 기능** | ✅ (도시/동네 비교) |
| **가격** | 무료 |

Niche의 핵심 교훈: **정량 데이터 + 주민 체감**을 결합
- 범죄 통계만으로는 "체감 안전도"를 못 잡음 → 주민 리뷰로 보완
- 안전귀가Navi에 적용: 향후 사용자 리뷰/별점 기능 추가 가능

#### NeighborhoodScout — 데이터 깊이 최고

| 항목 | 내용 |
|---|---|
| **범죄 데이터** | 60개+ 세부 통계, 900만+ 건, 98% 정확도 |
| **총 데이터** | 2,723개 고유 요소 |
| **세분화** | 센서스 블록 단위 (가장 작은 단위) |
| **비교 기능** | ✅ |
| **가격** | B2C $49.99~$119.99/월, B2B 연 $5,000+ |

NeighborhoodScout의 핵심 교훈: **데이터 세분화가 경쟁력**
- "관악구"가 아니라 "봉천동 OO길" 단위로 안전을 보여줘야 의미 있음

### 3.2 미국 부동산 앱의 범죄 데이터 제거 — 한국에는 기회

| 시점 | 사건 |
|---|---|
| 2021.11 | Redfin CEO, "범죄 데이터가 인종 편향을 강화한다" 공식 선언 |
| 2021.12 | Trulia (Zillow 소유), 범죄 데이터 삭제 |
| 2022~ | Realtor.com도 동반 제거 |

**제거 이유:**
- 미국의 레드라이닝(Redlining) 역사: 흑인 밀집 지역을 "위험"으로 낙인찍어 대출/보험 차별
- 공정주거법(Fair Housing Act) 위반 우려
- 범죄 데이터가 사실상 인종/소득 프록시로 작동

**한국에 이 문제가 적용되지 않는 이유:**
- 한국에는 미국식 인종 기반 주거 차별(레드라이닝)의 역사가 없음
- 서울시 자치구 간 인종 구성 차이가 거의 없음
- 따라서 범죄 데이터 공개가 특정 인종 차별로 이어질 리스크가 구조적으로 낮음
- → 미국에서 불가능해진 "부동산+범죄데이터 결합"이 한국에서는 가능

**단, 주의할 리스크:**
- "특정 동네 낙인 효과" — 범죄 많은 동네 집값 하락 우려
- 대응: 절대 점수보다 **상대 비교**, **개선 추세** 함께 표시, 서울시 평균 대비 표현

## 4. 학술/실무 검증된 안전 변수 체계

### 4.1 CPTED (범죄예방 환경설계) 5대 원리

| 원리 | 측정 가능 변수 | 안전귀가Navi 데이터 소스 |
|---|---|---|
| **자연적 감시** | CCTV 밀도, 가로등 밀도, 건물 1층 상가 비율, 조경 시야 확보 | CCTV API, 가로등 API |
| **접근통제** | 골목 진입로 수, 도로 연결성, 막다른 골목 여부 | 보행자 도로 데이터, OSM |
| **영역성 강화** | 담장/펜스 유무, 공적/사적 공간 구분 | (직접 측정 어려움) |
| **활동 활성화** | 상가 밀도, 유동인구, 편의점 | 생활인구 데이터, 편의점 데이터 |
| **유지관리** | 건물 노후도, 쓰레기/낙서 | (직접 측정 어려움) |

**서울시 CPTED 적용 실적**: 5개 지역 적용 결과 **5대 범죄 최대 54% 감소**

### 4.2 범죄예방건축기준 고시 (국토교통부) — 법적 기준

| 구분 | 기준 | 안전귀가Navi 반영 가능성 |
|---|---|---|
| CCTV | 주차장, 승강기, 옥상 출입구 설치 의무 | 건물별 CCTV 유무는 데이터 한계 |
| **조명** | 출입구 300lx, 보행통로 50lx, 주차 10lx | 가로등 밀도로 프록시 가능 |
| 출입통제 | 비밀번호/카드키 | (개별 건물 데이터 한계) |
| 조경 | 사각지대 최소화 | (위성/도로뷰로 향후 가능) |

### 4.3 주거만족도 조사의 안전 변수 (국토교통부)

| 변수 | 유형 | 측정 방식 |
|---|---|---|
| 방범 상태 | 객관적 | CCTV, 경비원, 방범창 유무 |
| 재난/화재 안전성 | 객관적 | 소방서 거리, 건물 구조 |
| **범죄 불안감** | 주관적 | 설문 (1~5점) |
| **야간 보행 안전감** | 주관적 | 설문 (1~5점) |

> 1인가구 대상 연구: **범죄 불안감이 주거 만족도에 유의미한 부(-)의 영향** — 불안할수록 만족도 하락

### 4.4 해외 Neighborhood Safety Index (학술)

**미국 NSSI (Neighborhood Sentiment and Safety Index)**:
- 방법론: 주성분분석(PCA)
- 객관적 변수: 범죄율, 빈곤가구 비율, 실업률, 과밀주거 비율
- 주관적 변수: 체감 안전도, 소음, 청결도, 미관, 동네 평판

## 5. 안전귀가Navi 주거 안전 스코어 재설계

### 5.1 기존 설계의 한계 (service-design.md)

기존 7개 변수(치안/CCTV/가로등/대피/유흥/도로/시간)는 **"귀가 경로" 관점에 치우쳐** 있었음.
주거 환경 비교 관점에서는 더 넓은 변수가 필요.

### 5.2 재설계: 2축 분리 스코어링

```
안전귀가Navi 안전 점수 체계
│
├── 🏠 주거 안전 점수 (Housing Safety Score) — 집 비교용
│   │
│   ├── [치안 안전] (30%)
│   │   ├── 반경 500m 5대 범죄 발생 건수 (자치구→동 추정)
│   │   ├── 범죄 유형별 가중치 (성범죄 > 절도)
│   │   └── 전년 대비 증감 추세
│   │
│   ├── [감시 인프라] (25%)
│   │   ├── CCTV 밀도 (대수/면적)
│   │   ├── CCTV 커버리지 추정 (사각지대 면적)
│   │   └── 가로등 밀도 + 점등 시간
│   │
│   ├── [생활 안전 인프라] (20%)
│   │   ├── 최근접 편의점 거리 (24시간 영업 여부)
│   │   ├── 최근접 경찰서/파출소 거리
│   │   ├── 최근접 소방서/병원 거리
│   │   └── 여성안심지킴이집 거리
│   │
│   ├── [환경 리스크] (15%)
│   │   ├── 유흥업소 밀집도 (반경 300m)
│   │   ├── 유동인구 야간 감소율
│   │   └── 골목길 사각지대 (도로 폭 < 4m 비율)
│   │
│   └── [주거 환경 품질] (10%)
│       ├── 건물 노후도 (건축연도)
│       ├── 반경 내 공원/녹지 비율
│       └── 대기질 (미세먼지 연평균)
│
└── 🚶 귀가 안전 점수 (Route Safety Score) — 경로 비교용
    │   (기존 service-design.md의 경로 알고리즘과 동일)
    ├── 경로 상 CCTV 커버율
    ├── 경로 상 가로등 밀도
    ├── 경로 상 편의점/대피시설
    ├── 경로 상 유흥업소 회피
    └── 시간대별 유동인구
```

### 5.3 주소 비교 기능 설계

**AreaVibes 방식 벤치마킹**: 최대 3곳 주소를 나란히 비교

```
┌─────────────────────────────────────────────────────┐
│  📊 주소 비교                           [주소 추가+] │
│                                                      │
│           봉천동 OO길 12   신림동 XX로 5   낙성대 YY  │
│  ─────────────────────────────────────────────────── │
│  종합      68점 🟡        78점 🟢        74점 🟢     │
│  ─────────────────────────────────────────────────── │
│  치안      ██░░ 62       ████ 82        ███░ 71      │
│  CCTV     ████ 85       ███░ 74        ████ 80      │
│  가로등    ██░░ 58       ████ 81        ███░ 72      │
```

## 6. "다방 안전 정보 서비스"와의 차별화 전략

다방이 이미 안전 관련 기능을 갖고 있으므로, **"다방이랑 뭐가 다른데?"**에 명확히 답해야 한다.

| 비교 축 | 다방 안전 정보 (2019~) | 안전귀가Navi |
|---|---|---|
| **데이터 범위** | CCTV, 경찰서, 파출소, 여성안심지킴이집 (4종) | CCTV + 가로등 + 범죄 + 유흥업소 + 편의점 + 도로 + 유동인구 + 기상 (8종+) |
| **보여주는 것** | 시설 **위치 핀** | 시설 위치 + **종합 안전 점수** + **AI 해석 리포트** |
| **범죄 데이터** | ❌ 없음 | ✅ 5대 범죄 발생 현황 + 추세 |
| **비교 기능** | ❌ 없음 | ✅ 최대 3곳 비교 + AI 코멘트 |
| **시간대 반영** | ❌ 없음 | ✅ 시간대별 안전도 변화 (주간/야간/심야) |
| **개인화** | ❌ 없음 | ✅ 우선순위 설정 (프리셋 + 커스텀) |
| **귀가 경로** | ❌ 없음 | ✅ 안전 귀가 내비 (완전히 다른 축) |
| **AI 활용** | ❌ 없음 | ✅ ML 스코어링 + 생성형 AI 리포트 + 경로 최적화 |

**한 문장 차별화:**
> 다방은 "경찰서가 어디 있는지" 알려주고, 안전귀가Navi는 "이 집이 얼마나 안전한지, 왜 그런지, 대안은 뭔지"까지 AI가 판단해준다.

```
          ──────────────────────────────────
```

**안전귀가Navi의 포지션:**
- 데이터: NeighborhoodScout 수준의 다변수 (한국 공공데이터 8종+)
- AI: SaferHood AI 수준의 AI 리포트 + 경로 최적화 (SaferHood에 없는 기능)
- + 귀가 내비 (어떤 해외 서비스에도 없는 기능 결합)

## 8. 발표 시 예상 질문 & 대응

### Q1: "다방이 이미 비슷한 걸 하고 있는데?"

> "다방의 안전 정보 서비스는 치안 시설의 위치를 지도에 표시하는 수준입니다. 안전귀가Navi는 8종 이상의 공공데이터를 ML로 종합해서 '이 집이 100점 만점에 몇 점인지'를 산출하고, 생성형 AI가 '왜 이 점수인지, 대안은 뭔지'까지 설명합니다. 시설 위치 표시와 AI 안전 판단은 차원이 다릅니다."

### Q2: "범죄 데이터 공개하면 특정 동네 낙인 찍히지 않나?"

> "미국에서는 인종 차별(레드라이닝) 우려로 Zillow/Redfin이 범죄 데이터를 제거했습니다. 하지만 한국에는 이 구조적 문제가 없습니다. 저희는 절대 순위 대신 서울시 평균 대비 상대 비교를 사용하고, '전년 대비 개선 추세'를 함께 보여줘서 동네 낙인 효과를 최소화합니다."

### Q3: "안심이 앱이랑 뭐가 달라?"

> "안심이 앱은 '관제센터가 CCTV로 감시해주는' 수동적 보호입니다. 안전귀가Navi는 '시민이 직접 데이터를 보고 집과 경로를 판단하는' 능동적 의사결정 도구입니다. 안심이 앱이 112 신고 연결이라면, 저희는 그 전 단계인 '위험을 미리 회피하는 예방'에 집중합니다."

### Q4: "공공데이터만으로 정확한 안전 점수가 가능한가?"

> "현재 열린데이터광장에서 CCTV, 가로등, 범죄, 유흥업소, 생활인구 등 8종 이상의 데이터를 API로 제공합니다. CPTED(범죄예방 환경설계) 연구에서 검증된 변수인 자연적 감시(CCTV, 가로등), 활동 활성화(유동인구, 상가), 접근통제(도로 구조) 모두 공공데이터로 측정 가능합니다. 서울시 CPTED 적용 지역에서 5대 범죄가 최대 54% 감소한 실증 결과가 이 변수들의 유효성을 뒷받침합니다."

### Q5: "수익화가 가능한가?"

> "글로벌 개인 안전 앱 시장은 $18억(2024)에서 $77억(2034)으로 CAGR 15.6% 성장 중입니다. Life360은 비슷한 프리미엄 모델로 $4.9억 매출을 달성했고, SaferHood AI는 리포트 건당 $10~35로 수익화하고 있습니다. 저희는 B2C(리포트+구독) + B2B(부동산 앱 API)의 이중 수익 모델을 설계했습니다."
