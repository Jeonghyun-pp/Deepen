# Deepen 기능 정의서 — 사용자에게 뭘 주는가

> 작성일: 2026-04-14
> 범위: 알고리즘·내부 파이프라인 제외. **"사용자가 화면에서 무엇을 받는가"** 관점의 기능 정의.
> 구성: (1) 지금 쓸 수 있는 것 / (2) 약속하고 있는 것 — 두 섹션으로 분리.

---

## 1. 현재 구현된 기능 (프로토타입에서 실제 동작)

### 1.1 지식 그래프 캔버스
**User story**
> "여러 자료에서 나온 개념·논문·질문을 한 화면에서 **공간적으로** 보고 싶다. 리스트로 나열된 건 관계가 안 보인다."

**Value prop**: 노드 38개 수준의 타입드 그래프(paper / concept / technique / application / question / memo / document)를 한 캔버스에 띄우고, 엣지 타입(citation / contains / 기타)을 시각적으로 구분해서 보여준다. **"자료들의 지형"을 한눈에** 준다.

**주는 것**
- 레이어드 DAG 기본 레이아웃 (force 튜닝 완료)
- 노드 타입별 크기·색상 차등
- 엣지 3단계 시각 hierarchy
- Projection 탭 전환 (그래프를 다른 관점으로 투사)

---

### 1.2 Roadmap Overlay — 로드맵 일급 객체
**User story**
> "그래프가 예쁜 건 알겠는데, **어디서부터 읽어야 하는지** 모르겠다."

**Value prop**: 그래프 위에 학습 순서를 오버레이로 얹어, "탐색용 캔버스"와 "순서가 있는 경로"를 같은 공간에서 전환시킨다. 유저는 **공간적 이해 ↔ 순차적 이해** 사이를 한 클릭으로 오간다.

---

### 1.3 캔버스 탭 시스템 (Doc / Note)
**User story**
> "그래프만으론 부족하다. 원문을 읽거나 내 생각을 메모할 공간이 같은 워크스페이스에 있어야 한다."

**Value prop**: 한 워크스페이스 안에 Doc 탭(원문 뷰 + projection section)과 Note 탭(플로팅 메모)을 공존시켜, 자료 → 그래프 → 메모가 **탭 전환만으로** 연결된다.

---

### 1.4 에이전트 채팅 + Tool Loop
**User story**
> "LLM이 내 그래프를 **실제로 건드릴 수** 있어야 의미가 있다. 일반 챗봇처럼 답만 주는 건 소용없다."

**Value prop**: 오른쪽 패널의 ChatPanel에서 에이전트에게 지시하면, 에이전트가 그래프를 조회·조작하는 6개 툴을 호출한다. 도구 호출은 ToolCallCard로 투명하게 노출되고, 쓰기 작업은 ApprovalCard를 거쳐 **사용자 승인 후에만** 실행된다.

**현재 에이전트가 가진 6개 도구**
1. `query_graph` — 그래프 조회
2. `find_path` — 두 노드 간 경로 탐색
3. `extract_concepts` — 텍스트에서 개념 추출
4. `search_papers_openalex` — OpenAlex 논문 검색
5. `add_node` — 노드 추가 (사용자 승인 필요)
6. `add_edge` — 엣지 추가 (사용자 승인 필요)

---

### 1.5 우측 4탭 패널 (Detail / Chat / Filter / …)
**User story**
> "선택한 노드의 상세, 에이전트 대화, 필터링이 화면 전환 없이 한 자리에 있어야 한다."

**Value prop**: NodeDetailPanel · ChatPanel · FilterBar · GapSummary 등을 한 패널 안 탭으로 묶어, **선택 → 대화 → 조작**을 한 흐름으로 수행.

---

### 1.6 상호작용 모델
- **Click**: 노드 선택 → 상세 표시
- **Shift+Click**: 다중 선택 / 범위 지정
- **Hover**: 프리뷰 툴팁
- Gap Summary, Concept Timeline, Export 모달 등 보조 뷰

---

### 1.7 랜딩·온보딩
- 베타 가입 폼, Deepy 가이드 캐릭터, Step/Problem/Solution/FAQ 섹션 등 **제품 약속을 전달하는 랜딩**이 가동 중.

---

## 2. 계획 중인 기능 (약속했지만 아직 안 준 것)

> 북극성: **"강의안 업로드 → 모든 내용이 노드로 분화 → 순서대로 학습 → '강의안을 빠짐없이 안다'는 것을 증명받는다."**

### 2.1 완결성 증명 — "여기까지 하면 끝"
**User story**
> "87페이지 강의안을 다 이해한 건지, 아닌지를 **확실히** 알고 싶다. NotebookLM도 ChatGPT도 이 약속을 안 한다."

**Value prop**: 원문의 모든 chunk(문단·수식·figure·표)가 어느 노드로 매핑되었는지 추적하고, 미매핑 블록을 숨기지 않고 드러낸다. 유저는 **"커버리지 94%, 미매핑 4개"** 같은 정직한 상태를 본 뒤, 미매핑 블록을 직접 승인해 100%를 만든다.

**주기로 한 것**
- Chunk → Node 역추적 링크
- 커버리지 배지 (100%일 때만 "완결")
- 미매핑 블록 리스트와 원클릭 승격
- 원문 증거 클릭 (모든 주장 옆에 증거)

---

### 2.2 Prerequisite DAG — 학습 순서의 지도
**User story**
> "개념 A를 읽다가 막혔을 때, **어디로 돌아가야 하는지** 자료가 안 알려준다. 목차는 저자 순서고, 학습 순서가 아니다."

**Value prop**: 개념 간 "이해 불가 관계"를 엣지로 그린 DAG를 자동 생성하고, 레이어별로 정렬해 **어디서 시작해 어디로 갈 수 있는지**를 보여준다. 저자의 목차 순서는 토글로만 제공.

**주기로 한 것**
- `logical-prerequisite` vs `pedagogical-order` 엣지 이원화
- 레이어드 로드맵 뷰 (Level 1 → 2 → 3)
- 사이클 감지 & axiom 후보 제안
- "이미 알아요" 버튼으로 노드 스킵 + 재검증

---

### 2.3 이해 확인 루프 — "읽었다"가 아닌 "증명했다"
**User story**
> "하이라이트하고 넘어가면 공부한 것 같지만, 시험장에선 기억 안 난다. **행동으로 증명**하는 단계가 필요하다."

**Value prop**: 노드 타입별 check item(cloze / 순서배열 / 객관식 / 논증 재구성)을 자동 생성하고, `unseen → viewed → tested → mastered` 상태 머신으로 진행도를 관리한다. 틀리면 prerequisite 역추적 경로를 제안.

**주기로 한 것**
- 노드별 자동 퀴즈 생성
- Mastery 상태 머신 + 재검증 스케줄
- 실패 시 "어느 prerequisite가 문제였나" 역추적 제안
- 논증형 채점 (문과·인문학 대응)

---

### 2.4 완주 뱃지 — 약속의 코드화
**User story**
> "이 강의안, 정말 다 한 건가? 누가 증명해 주나?"

**Value prop**: `coverage == 100% AND ∀ node mastered AND DAG acyclic` 불변식을 DB 레벨에서 강제하고, 통과 시에만 **"이 강의안(p.1~87, 54개념)을 빠짐없이, 순서대로 이해했습니다"** 뱃지 발급.

---

### 2.5 멀티모달 Chunk
**User story**
> "강의안엔 수식·figure·표가 본문만큼 중요하다. 텍스트만 처리하면 절반만 공부한 것."

**Value prop**: PDF를 text / equation / figure / table 타입으로 분리해 각각을 독립 chunk로 관리. Figure는 OCR + 캡션 + 주변 텍스트 복합 chunk.

---

### 2.6 사용자 반박 권한
**User story**
> "AI가 틀린 매핑을 만들었을 때, 내가 고칠 수 없으면 그건 내 도구가 아니다."

**Value prop**: 모든 LLM 제안(엣지, chunk 매핑, 퀴즈)은 `proposed_*` 상태로 저장되고, 사용자 승인·수정·거부가 1급 액션. 반박 시 불변식 재계산 & 뱃지 동적 갱신.

---

### 2.7 타깃 확장 시나리오 (로드맵)
- 대학원 수업 번들 (한 과목 누적)
- 연구자 논문 읽기 그래프 (v2 원래 DNA)
- 전문시험 (CFA·국시·변호사) — "대충 요약 금지" 시장
- B2B 사내 교육·컴플라이언스

---

## 3. 한눈에 보는 포지셔닝

| | 경쟁 상대 | Deepen 약속 |
|---|---|---|
| vs **유니브 AI** | "더 빨리" | **"빠짐없이"** |
| vs **NotebookLM** | "자료를 더 잘 소비" | **"완전히 이해했음을 증명"** |
| vs **ChatGPT 업로드** | "무엇이든 질문" | **"상태 있는 학습 여정"** |

**메타 약속**
> Deepen은 학습자를 대신해 공부해주지 않는다. 학습자가 **"자기가 어디까지 이해했는지 정확히 아는 상태"** 를 만들어준다.
