## Deepen — 유형 단위로 약점을 추적하는 입시 학습 코치

> *EBS를 수십 번 돌려도 안 풀리던 게, 강사 5분이면 풀린다.*
> 그 5분을 살 수 있는 학생과 없는 학생이 매일 갈라진다.

### 1. 어떤 문제를 해결하는가

한국 입시 수험생의 진짜 약점은 **단원이 아니라 유형**, 그리고 **이전 학년의 결손**에 있다. 고2 미분 곡선 밖 접선을 못 푸는 진짜 이유는 흔히 중3 판별식이다. 강사 5분이면 보이지만, 학생도 일반 챗봇도 모른다.

기존 학습 서비스는 **단원 단위 진단·맞음/틀림 이분법·정적 개념 지도**에 머문다. 결과:

- 소득 상하위 가구 사교육비 **133배 격차** (하위 10% 월 3,042원 vs 상위 10% 40만 6,986원, 통계청 2024)
- 강남·서초 학업중단율 **2.7%** — 부유한 동네도 시스템에서 도망친다 (경향신문 2025-08)
- 5년간 자퇴자 누적 **+89%** (14,140 → 26,753명, 교육부)
- 정답이지만 헷갈리던 문제가 **실제 시험 변별력**을 결정하는데 기존 채점은 이걸 놓친다

수험생은 ‘이제 뭘 해야 하지?’를 매일 다시 고민한다. AI 시대에 지식은 싸졌지만 **학습권**은 더 비싸졌다. 동시에 시장은 자라고 있다 — 수능 응시자 **46만 3,486명**(2025학년), 한국 초중고 사교육비 **TAM 29.2조원** (+7.7% YoY, 통계청), 수험생 1% × 월 3-5만원 = **SOM 18-30억** (3년 침투). 사회적 임팩트와 BM 모멘텀이 같은 도메인 안에 있다.

### 2. 어떻게 해결하는가 — Deepen

PDF 강의안·평가원 기출을 한 장 업로드하면 5분 안에 (1) **Parse** — 표·수식·도표를 보존한 채 구조화하고, (2) **Map** — Concept(개념)–Pattern(유형)–Item(기출) 3계층 학습 지도를 자동 생성한다. 학생은 PDF 위에 펜으로 풀이를 작성하고(tldraw 단일 캔버스, 종이처럼 잉크가 배경에 올라탄다) Vision이 단계별 LaTeX와 5지선다 답을 자동 인식한다. 채점은 **3분기** — 정답·오답·**정답이지만 헷갈림**. 오답은 *현재 개념 부족·유형 인식 실패·접근 방향 오류·계산 실수·조건 해석 오류·그래프 해석 오류·풀이 논리 비약·이전 학년 선행 결손* 8가지 기준으로 원인을 추적하고, 막히면 prerequisite 그래프를 거꾸로 타고 진짜 결손 후보를 자동 도출 — 1-3분짜리 **리캡 카드**로 메운 뒤 원래 문제로 복귀시킨다.

차별점은 7가지 축에서 동시에 다르다. 어떤 경쟁사(콴다·메가스터디·NotebookLM·유니브 AI·매쓰홀릭)도 이 중 4개 이상 동시에 갖지 못한다:

1. 단원 → **유형(Pattern) 단위** 약점 추적
2. 현재 학년 → **이전 학년 결손 역추적**
3. 맞음·틀림 → **‘정답이지만 헷갈림’ 별도 분류**
4. 정적 개념 지도 → **풀이 결과가 반영되는 동적 학습 지도**
5. 일반 챗봇 → **문제·유형·선행·이력을 모두 아는 코치 AI**
6. 단원 기반 추천 → **유형 집합·조합·응용 깊이 기반 추천**
7. 5분짜리 강사 → **AI로 모든 학생에게 동일 비용 제공**

내부 자체 측정: PDF→그래프 자동 생성 파이프라인에서 노이즈 감소 전략 적용으로 정밀도 **60.5% → 74.3%** (재무관리 도메인 베이스라인 검증).

### 3. 왜 Upstage 제품으로 해결 가능한가

해커톤에서 통합하는 Upstage 제품은 **3개**, 모두 우리 코드베이스의 정확한 갈아끼울 지점이 있다.

| Upstage 제품 | 대체할 자체 컴포넌트 | 이유 |
|---|---|---|
| **Document Parse** | `unpdf` 휴리스틱 분할 (`lib/pipeline/parse-pdf.ts`) | TEDS 94.48 / 3.79s/page. 한국 평가원 기출의 **표·수식·도표를 보존**한 채 HTML/Markdown 변환 — 현재 placeholder로 버려지던 정보를 살려 분모(`allChunks`) 정확도를 끌어올린다. Layout-aware parsing이 우리 노이즈 감소 4-layer 플랜의 Layer 1 정확히 그것. |
| **Information Extract** | OpenAI gpt-4o-mini 노드 추출 (`extract-nodes.ts`) + chunk-mapping placeholder (`chunk-mapping.ts`) | 스키마 기반 + **한국어 강세** + 범용 LLM 대비 정확도 +10%. ‘Concept–Pattern–Item’ 추출은 도메인 스키마가 분명해서 Information Extract의 강점(의도·암시 정보까지 잡아냄)이 정확히 맞는다. |
| **Document Classify** | 없음 (현재 확장자 검증만, `app/api/documents/upload/route.ts`) | 업로드 PDF를 `{기출, 강의안, EBS, 노트}`로 자동 분류 → 강의안/기출 파이프라인 자동 분기. 모델 학습 없이 100+ 클래스 지원이므로 새 카테고리 추가 비용 0. |

세 제품을 통합한 결과:

- **확률적 답변이 아닌 결정론적 진단**: Document Parse·Classify는 결정론적 layer, Information Extract는 스키마 기반 — 노이즈가 최소화된다 (자체 측정 노이즈 감소 60.5% → 74.3%의 다음 단계).
- **비용·레이턴시 절감**: 자체 측정 기준 doc 1건당 OpenAI 대비 ~30~50% 절감 예상 (Day 1 사전 검증 후 정확 수치 확정).

### 4. 데모 시나리오 (5분)

심사위원·다른 팀이 직접 만져볼 수 있게 한다:

1. 평가원 기출 1세트 + 강의안 1세트를 드래그앤드롭으로 업로드
2. 30초 내에 Concept–Pattern–Item 그래프가 그려짐
3. 학생이 풀이를 펜으로 입력 → 오답
4. ‘진짜 결손은 중3 판별식’ — prerequisite 노드 하이라이트 + 2분짜리 리캡 카드 생성
5. 원래 문제로 복귀 → 정답
