# Deepen 교육 서비스 UX 플로우차트

```text
[1. 과목 / 시험 선택]
→ 사용자는 먼저 학습하고자 하는 과목(수학 / 과학탐구 등)과 시험 범위(수능 / 6모 / 9모 / 최근 5개년)를 선택한다.
→ 이 단계는 단순한 과목 선택이 아니라, 이후 그래프에 어떤 기출 데이터와 중요도가 반영될지를 결정하는 시작점이다.

        ↓

[2. 기출 기반 개념 지식 그래프 화면 진입]
→ 선택한 과목과 시험 데이터에 맞는 개념 지식 그래프가 나타난다.
→ 이 그래프는 단순한 개념 구조도가 아니라, 최근 기출에서 실제로 어떤 개념이 중요하게 출제되었는지를 시각적으로 보여주는 핵심 화면이다.
→ 예를 들어 빈출 개념, 킬러 문항 관련 개념, 정답률이 낮은 문제와 연결된 개념 등이 색상, 굵기, 테두리 등으로 강조된다.

        ↓

[3. 중요 개념 탐색 및 특정 개념 노드 선택]
→ 사용자는 그래프를 탐색하면서 어떤 개념이 중요한지, 어떤 개념이 다른 개념과 연결되는지 확인할 수 있다.
→ 이때 특정 개념 노드를 클릭하면, 해당 개념이 실제 시험 문제와 어떻게 연결되는지를 확인하는 다음 단계로 넘어간다.
→ 즉, 그래프는 단순 시각화가 아니라 “무엇을 공부해야 하는지”를 안내하는 네비게이션 역할을 한다.

        ↓

[4. 개념 상세 화면 진입]
→ 선택한 개념에 대한 상세 정보가 제공된다.
→ 여기에는 개념 요약, 시험에서의 의미, 최근 출제 이력, 관련 기출 문제 수, 난이도별 문제 분포 등의 정보가 포함된다.
→ 사용자는 이 화면을 통해 “이 개념이 왜 중요한지”, “실제 시험에서 어떤 방식으로 출제되는지”를 이해할 수 있다.

        ↓

[5. 난이도별 연관 문제 확인]
→ 개념 상세 화면에서는 해당 개념과 연결된 문제들이 난이도별로 정리되어 제공된다.
→ 예를 들어 쉬움 / 중간 / 어려움 / 킬러 문항으로 구분하여 문제를 배치할 수 있다.
→ 이를 통해 사용자는 자신의 현재 실력에 맞는 문제부터 풀 수 있고, 점진적으로 더 어려운 문제로 올라갈 수 있다.
→ 이 단계는 “개념을 아는 것”에서 “문제에 적용하는 것”으로 넘어가는 핵심 구간이다.

        ↓

[6. 문제 선택 및 문제 풀이]
→ 사용자는 원하는 문제를 선택하여 실제로 문제를 푼다.
→ 이 과정에서 단순히 정답만 제출하는 방식도 가능하고, 필요하다면 풀이과정까지 함께 기록하도록 확장할 수 있다.
→ 향후에는 풀이 시간, 힌트 사용 여부, 사용자의 풀이 방식 등도 함께 수집하여 더 정교한 분석이 가능하다.

        ↓

[7. 채점 및 결과 분기]
→ 문제 제출 후 시스템은 정답 여부를 판별하고, 결과에 따라 사용자를 다른 흐름으로 안내한다.
→ 이 단계는 단순히 “맞음 / 틀림”을 보여주는 것이 아니라, 이후 학습 방향을 나누는 분기점 역할을 한다.

        ↓

 ┌───────────────────────────────┬───────────────────────────────┬───────────────────────────────┐
 │                               │                               │                               │
[7-A. 정답]                  [7-B. 오답]                    [7-C. 정답이지만 헷갈림]
→ 문제를 정확히 풀었고        → 문제를 틀린 경우에는            → 정답은 맞췄지만 확신이 없거나,
개념 적용도 적절했던 경우       왜 틀렸는지 원인을 분석한다.        풀이가 불안정했던 경우를 따로 구분한다.
다음 문제로 넘어갈 수 있다.     단순 오답 여부보다                이는 실제 시험에서 다시 틀릴 가능성이 높은
                               “개념 부족인지, 계산 실수인지,     문제를 잡아내기 위한 중요한 분류이다.
                               접근 방식 오류인지”를 구분하는     따라서 이 문제는 별도로 저장하여
                               것이 중요하다.                    복습 대상으로 관리할 수 있다.
 └───────────────────────────────┴───────────────────────────────┴───────────────────────────────┘

        ↓

[8. AI 해설 / 질의응답 / 오답 분석]
→ 사용자는 문제 풀이 후 AI와 상호작용할 수 있다.
→ 오답인 경우에는 왜 틀렸는지, 어떤 개념이 부족한지, 어떤 단계에서 막혔는지에 대해 질문할 수 있다.
→ 정답을 맞힌 경우에도 “왜 이 방식으로 풀리는지”, “다른 풀이도 가능한지”, “내 풀이가 안전한지”를 물어볼 수 있다.
→ 이 단계에서 AI는 단순 해설 제공자가 아니라, 사용자의 이해 상태를 분석하고 부족한 부분을 보완해주는 코치 역할을 한다.

        ↓

[9. 오답노트 / 헷갈린 문제 자동 저장]
→ 문제 결과는 자동으로 기록되어 오답노트 또는 헷갈린 문제 목록에 저장된다.
→ 단순히 틀린 문제를 모아두는 것이 아니라, 개념 오답 / 계산 오답 / 접근 방식 오류 / 맞았지만 불안정함 등의 유형으로 분류할 수 있다.
→ 이를 통해 사용자는 자신이 자주 틀리는 패턴을 확인할 수 있고, 단순 반복이 아니라 원인 중심 복습이 가능해진다.

        ↓

[10. 약점 개념을 그래프에 반영]
→ 문제풀이 결과를 바탕으로 사용자의 약점 개념이 다시 개념 지식 그래프에 반영된다.
→ 예를 들어 특정 개념에서 반복적으로 틀리면 해당 노드가 경고 색상으로 표시되거나, 관련 선행 개념까지 함께 강조될 수 있다.
→ 이 단계는 Deepen의 핵심 차별점 중 하나로, 문제풀이 결과가 다시 그래프 구조에 반영되면서 그래프가 “나만의 학습 상태 지도”로 변하게 된다.

        ↓

[11. 다음 학습 액션 추천]
→ 시스템은 사용자의 결과를 바탕으로 다음 행동을 추천한다.
→ 예를 들어,
   - 같은 개념의 쉬운 문제 다시 풀기
   - 관련 선행 개념 복습하기
   - 유사한 중간 난이도 문제 도전하기
   - 킬러 문항 대비용 확장 문제 풀기
  와 같은 형태로 추천이 가능하다.
→ 즉, 사용자는 매번 “이제 뭘 해야 하지?”를 고민하지 않아도 되고, 자연스럽게 다음 학습 흐름으로 이어질 수 있다.

        ↓

[12. 반복 학습 루프]
→ 사용자는 다시 그래프로 돌아가 새로운 개념을 선택하거나, 추천된 문제를 풀면서 학습을 반복한다.
→ 이 루프는 다음과 같이 요약된다.

   기출 기반 중요 개념 확인
   → 개념 선택
   → 관련 문제 풀이
   → 채점 및 분석
   → AI 피드백
   → 약점 개념 그래프 반영
   → 다음 문제/개념 추천
   → 반복

→ 결국 Deepen의 UX는 “그래프를 보는 서비스”가 아니라,
   “그래프를 기반으로 문제를 풀고, 분석하고, 다시 학습 방향을 추천받는 서비스”로 작동한다.
```
