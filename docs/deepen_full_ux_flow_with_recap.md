# Deepen 전체 UX 플로우
## 이전 학년 개념 리캡 기능 포함 버전

[1. 서비스 진입]
→ 사용자는 Deepen에 접속한다.
→ Deepen은 단순 개념 학습 서비스가 아니라, 입시 문제풀이를 중심으로 사용자의 약점 개념과 약점 유형을 추적하는 AI 학습 코치로 작동한다.
→ 첫 진입 화면에서는 사용자가 어떤 시험을 준비하는지, 어떤 과목과 단원을 학습할 것인지 설정한다.

        ↓

[2. 학습 목표 / 시험 범위 선택]
→ 사용자는 학습 목표와 시험 범위를 선택한다.
→ 입시교육에서는 단순 과목 선택보다 “어떤 시험을 대비하는가”가 중요하므로, 수능 / 6모 / 9모 / 최근 5개년 / 특정 연도 기출 등의 필터를 제공한다.

선택 항목 예시:
- 학년: 고1 / 고2 / 고3 / N수
- 과목: 수학 / 과학탐구
- 세부 단원: 수학Ⅱ > 미분
- 시험 범위: 최근 5개년 수능 + 6모 + 9모
- 학습 목표: 개념 복습 / 기출 풀이 / 약점 보완 / 킬러 대비 / 오답 복구

        ↓

[3. 개념-유형-문제 그래프 진입]
→ 사용자는 선택한 과목과 시험 범위에 맞는 그래프 화면에 진입한다.
→ 이 그래프는 단순 개념 지식 그래프가 아니라, Concept - Pattern - Item이 연결된 입시형 학습 그래프다.

그래프 계층:
- 상위 개념: 미분
- 하위 개념: 미분의 활용
- 유형(Pattern): 곡선 밖의 점에서 접선의 방정식
- 문제(Item): 2025 수능 N번, 2024 6모 N번 등

→ 여기서 핵심은 “개념”뿐 아니라 “유형”과 “문제”까지 연결된다는 점이다.
→ 입시 수학에서는 같은 개념도 여러 유형으로 나뉘기 때문에, 사용자의 실력은 개념 단위가 아니라 유형 단위로 추적되어야 한다.

        ↓

[4. 기출 기반 중요 개념·유형 시각화]
→ 최근 5개년 수능/모의고사에서 자주 등장한 개념과 유형이 그래프 위에 강조된다.
→ 빈출 개념, 킬러 문항 관련 유형, 정답률 낮은 문제와 연결된 유형이 색상, 굵기, 테두리 등으로 시각화된다.

표시 예시:
- 진한 색: 빈출 개념/유형
- 붉은 테두리: 킬러 또는 준킬러 유형
- 노란색 표시: 정답률 낮은 문제와 연결된 유형
- 회색 노드: 아직 학습하지 않은 개념/유형
- 초록색 노드: 사용자가 안정적으로 푸는 유형
- 경고 아이콘: 사용자의 반복 오답 유형

→ 이 화면에서 사용자는 “무엇을 공부해야 하는지”를 단순 목록이 아니라 그래프 구조로 확인한다.
→ 그래프는 단순 시각화가 아니라, 기출 중요도와 개인 약점을 함께 보여주는 입시 전략 지도 역할을 한다.

        ↓

[5. 이전 학년 선행 개념 연결 표시]
→ 현재 학습 중인 개념이나 유형과 연결된 이전 학년 개념도 그래프에 함께 표시된다.
→ 예를 들어 사용자가 ‘미분의 활용’ 단원을 보고 있다면, Deepen은 이 유형을 풀기 위해 필요한 중학교/고1/수학Ⅰ 개념까지 연결한다.

예시:
[중3: 이차방정식의 판별식]
        ↓
[고1: 함수와 그래프]
        ↓
[수학Ⅱ: 미분계수]
        ↓
[수학Ⅱ: 미분의 활용]
        ↓
[유형: 곡선 밖의 점에서 접선의 방정식]

→ 이 기능의 목적은 현재 문제를 못 푸는 이유를 현재 단원 안에서만 찾지 않는 것이다.
→ 사용자가 고2 미분 문제를 틀렸더라도, 실제 원인은 중학교의 판별식, 고1의 함수 대입, 일차함수의 기울기 개념일 수 있다.
→ Deepen은 이런 선행 개념 결손을 그래프에서 함께 보여준다.

        ↓

[6. 개념 또는 유형 노드 선택]
→ 사용자는 그래프에서 특정 개념 또는 유형 노드를 클릭한다.
→ 이때 선택 대상은 단순 개념뿐 아니라, 입시 문제풀이에 더 직접적인 유형 노드까지 포함한다.

예시:
- 개념 노드: 미분의 활용
- 유형 노드: 곡선 밖의 점에서 접선의 방정식
- 이전 개념 노드: 이차방정식의 판별식

→ 사용자가 특정 유형을 선택하면 Deepen은 해당 유형의 출제 방식, 관련 기출, 필요한 선행 개념, 사용자의 숙련도를 함께 보여준다.

        ↓

[7. 개념/유형 상세 화면 진입]
→ 선택한 노드의 상세 화면이 열린다.
→ 이 화면에서는 단순 설명보다 “실제 시험 문제와 어떻게 연결되는가”가 강조되어야 한다.

제공 정보:
- 개념/유형 요약
- 시험에서의 출제 방식
- 최근 출제 이력
- 관련 기출 문제 수
- 난이도별 문제 분포
- 연결된 선행 개념
- 연결된 이전 학년 개념
- 연결된 응용 유형
- 나의 숙련도
- 추천 학습 액션

예시:
[유형: 곡선 밖의 점에서 접선의 방정식]

시험에서의 의미:
- 접점을 변수로 두고 접선 조건을 세우는 유형
- 곡선과 직선이 한 점에서 만난다는 조건을 판별식 D=0 또는 접점 조건으로 해석하는 경우가 많음

필요 선행 개념:
- 접선의 의미
- 미분계수
- 함수값 대입
- 이차방정식의 판별식
- 일차함수의 기울기

        ↓

[8. 이전 학년 개념 리캡 필요 여부 확인]
→ Deepen은 사용자가 선택한 개념/유형을 학습하기 전에 필요한 이전 학년 개념을 점검한다.
→ 사용자의 과거 풀이 이력, 오답 유형, 리캡 퀴즈 결과, AI 질문 내용을 바탕으로 선행 개념 결손 가능성을 판단한다.

판단 기준:
- 해당 선행 개념과 연결된 문제를 자주 틀렸는가?
- 관련 유형에서 풀이 시간이 오래 걸렸는가?
- AI에게 해당 개념과 관련된 질문을 자주 했는가?
- 맞았지만 헷갈린 문제에 해당 개념이 반복 등장했는가?
- 리캡 퀴즈를 틀렸는가?

        ↓

┌────────────────────────────────────┬────────────────────────────────────┐
│ 이전 학년 개념 리캡 필요             │ 이전 학년 개념 리캡 불필요              │
└──────────────────┬─────────────────┴──────────────────┬─────────────────┘
                   ↓                                    ↓
[8-A. 짧은 리캡 카드 제공]              [8-B. 바로 문제풀이 단계로 이동]
                   ↓                                    ↓

[8-A. 짧은 리캡 카드 제공]
→ 사용자가 현재 문제나 유형을 이해하는 데 필요한 이전 개념을 1~3분 안에 복습할 수 있도록 리캡 카드를 제공한다.
→ 리캡은 긴 강의가 아니라, 현재 문제를 풀기 위한 최소 개념 복구에 초점을 맞춘다.

리캡 카드 구성:
- 개념명
- 왜 이 개념이 필요한가
- 3줄 핵심 설명
- 대표 예시 1개
- 확인 퀴즈 1개
- 원래 문제/유형으로 돌아가기 버튼

예시:
[리캡 카드: 이차방정식의 판별식]

왜 필요한가?
곡선 밖의 점에서 접선을 그을 때, 접선과 곡선이 한 점에서만 만나야 하므로 이차방정식의 중근 조건을 사용한다.

핵심 설명:
- 이차방정식 ax² + bx + c = 0의 판별식은 D = b² - 4ac이다.
- D = 0이면 서로 같은 두 실근, 즉 중근을 가진다.
- 접선 조건에서는 곡선과 직선이 한 점에서만 만나므로 D = 0을 자주 사용한다.

확인 퀴즈:
이차방정식 x² - 2kx + 9 = 0이 중근을 가지려면 k는 어떤 값을 가져야 하는가?

        ↓

[8-C. 리캡 퀴즈 / 미니 문제 풀이]
→ 사용자는 리캡을 읽은 뒤 매우 짧은 확인 문제를 푼다.
→ 목적은 완전한 단원 학습이 아니라, 현재 문제를 풀기 위한 선행 개념이 복구되었는지 확인하는 것이다.

결과:
- 통과: 원래 개념/유형 문제로 복귀
- 미통과: 더 쉬운 리캡 또는 기초 문제 추천

        ↓

[9. 난이도별 연관 문제 확인]
→ 사용자는 선택한 개념/유형과 연결된 문제를 난이도별로 확인한다.
→ 문제는 단순 단원 기준이 아니라 Pattern Node 기준으로 정리된다.

난이도 구조:
- 쉬움: 기본 개념 적용 문제
- 중간: 대표 유형 문제
- 어려움: 변형/응용 문제
- 킬러: 여러 유형이 결합된 고난도 문제

예시:
개념: 미분의 활용
유형: 곡선 밖의 점에서 접선의 방정식

쉬움:
- 접점을 변수로 두고 접선 구하기

중간:
- 외부 점 조건과 접선 조건 결합

어려움:
- 접선 조건 + 극값 조건 결합

킬러:
- 접선 조건 + 그래프 개형 + 실근 개수 판단

        ↓

[10. 문제풀이 모드 선택]
→ 사용자는 자신의 목적에 맞게 문제풀이 모드를 선택한다.

모드:
1. 연습 모드
   - 힌트 사용 가능
   - AI 질문 가능
   - 시간 제한 약함
   - 개념/유형 학습 중인 사용자에게 적합

2. 실전 모드
   - 힌트 없음
   - 제한 시간 있음
   - 제출 후 해설 확인
   - 수능 실전 감각 훈련에 적합

3. 챌린지 모드
   - 특정 유형을 연속으로 풀이
   - 점수/정답률/속도 반영
   - 게임화 요소를 통해 반복 학습 유도

4. 오답 복구 모드
   - 이전에 틀렸거나 헷갈린 문제 재풀이
   - 유사 문제와 함께 제공
   - 약점 회복에 적합

5. 리캡 후 재도전 모드
   - 이전 학년 개념 리캡을 본 뒤 원래 문제에 다시 도전
   - 선행 개념 결손 복구 여부 확인에 적합

        ↓

[11. 문제 풀이]
→ 사용자는 문제를 푼다.
→ 기본적으로 정답을 입력하고, 필요하면 풀이과정까지 제출할 수 있다.

수집 데이터:
- 정답 여부
- 풀이 시간
- 힌트 사용 여부
- AI 질문 여부
- 자신감 여부
- 맞았지만 헷갈렸는지 여부
- 풀이과정 이미지 또는 텍스트
- 문제 풀이 중 막힌 지점

→ 이 데이터는 이후 유형 숙련도, 개념 이해도, 이전 학년 개념 결손 판단에 모두 사용된다.

        ↓

[12. 채점 및 결과 분기]
→ 문제 제출 후 시스템은 단순히 맞고 틀림만 판단하지 않는다.
→ 결과는 세 가지로 나눈다.

1. 정답
2. 오답
3. 정답이지만 헷갈림

→ 이 중 “정답이지만 헷갈림”은 매우 중요하다.
→ 입시에서는 맞았지만 불안정한 문제도 실제 시험에서 다시 틀릴 가능성이 높기 때문이다.

        ↓

┌───────────────────────────────┬───────────────────────────────┬───────────────────────────────┐
│ [12-A. 정답]                   │ [12-B. 오답]                   │ [12-C. 정답이지만 헷갈림]       │
└───────────────┬───────────────┴───────────────┬───────────────┴───────────────┬───────────────┘
                ↓                               ↓                               ↓

[12-A. 정답]
→ 정답인 경우에는 다음 문제로 넘어갈 수 있다.
→ 다만 풀이 시간이 오래 걸렸거나 힌트를 사용했다면 완전 숙련으로 보지 않는다.
→ 정답 여부, 풀이 시간, 힌트 사용 여부, 자신감 여부를 함께 반영해 유형 숙련도를 업데이트한다.

가능한 피드백:
- “정답입니다. 다만 평균 풀이 시간보다 오래 걸렸습니다.”
- “이 유형은 맞혔지만 아직 속도 개선이 필요합니다.”
- “다음에는 같은 유형의 중간 난이도 문제를 추천합니다.”

        ↓

[12-B. 오답]
→ 오답인 경우에는 오답 원인을 분석한다.
→ 단순히 해설을 보여주는 것이 아니라, 어떤 개념/유형/선행 개념에서 막혔는지 추적한다.

분석 기준:
- 현재 개념 부족
- 유형 인식 실패
- 접근 방향 오류
- 계산 실수
- 조건 해석 오류
- 그래프 해석 오류
- 풀이 과정 논리 비약
- 이전 학년 선행 개념 결손

예시:
“이 문제는 미분 계산 자체보다, 곡선 밖의 점에서 접선을 그을 때 접점을 변수로 두는 발상을 떠올리는 것이 핵심입니다. 또한 접선과 곡선이 한 점에서 만난다는 조건을 판별식 D=0으로 연결하는 과정이 필요합니다.”

        ↓

[12-C. 정답이지만 헷갈림]
→ 정답은 맞췄지만 사용자가 헷갈렸다고 표시한 문제는 별도 저장한다.
→ 이 문제는 오답은 아니지만 약점 후보로 분류한다.
→ 풀이 시간이 길거나, 힌트를 사용했거나, AI 질문이 많았다면 숙련도는 낮게 반영한다.

가능한 피드백:
- “정답이지만 풀이 확신도가 낮은 문제로 저장되었습니다.”
- “이 문제는 다시 풀었을 때 틀릴 가능성이 있으므로 유사 문제 2개를 추천합니다.”
- “현재 이 유형은 숙련도는 중간이지만 안정성이 낮습니다.”

        ↓

[13. AI 해설 / 질의응답]
→ 사용자는 문제 풀이 후 AI에게 질문할 수 있다.
→ AI는 일반 챗봇이 아니라 현재 문제, 연결된 개념, 연결된 유형, 이전 학년 선행 개념, 사용자의 풀이 결과를 알고 있는 문제풀이 코치여야 한다.

가능 질문:
- “왜 여기서 접점을 t로 둬야 해?”
- “왜 판별식을 써야 해?”
- “내 풀이가 왜 틀렸어?”
- “이 문제 다른 풀이 있어?”
- “이 유형은 어떻게 알아차려?”
- “내 풀이가 답지랑 다른데 괜찮아?”
- “이전에 배운 어떤 개념이 필요한 거야?”
- “비슷한 문제 더 풀고 싶어.”

→ AI는 답변할 때 현재 문제의 유형과 선행 개념을 함께 설명한다.
→ 예를 들어 “이건 미분 문제가 아니라, 미분 + 판별식 + 접선 조건이 결합된 유형”이라고 설명할 수 있어야 한다.

        ↓

[14. 풀이과정 교정]
→ 사용자가 아이패드나 종이에 쓴 풀이과정을 업로드하면 OCR 또는 이미지 인식으로 풀이를 분석한다.
→ 초기 MVP에서는 완전 자동 분석보다, AI가 풀이를 요약하고 사용자가 확인하는 방식이 현실적이다.

흐름:
1. 풀이 이미지 업로드
2. AI가 풀이 단계 인식
3. 사용자에게 “이 풀이 요약이 맞나요?” 확인
4. 오류 가능성이 높은 줄 표시
5. 공식 해설 또는 대체 풀이와 비교
6. 교정 피드백 제공

가능한 피드백:
- “접근은 맞지만 두 번째 줄에서 부등호 방향이 바뀌지 않았습니다.”
- “이 단계에서 그래프의 대칭성을 사용했지만, 문제 조건상 대칭성이 보장되지 않습니다.”
- “정답은 맞았지만 이 풀이는 일반성이 부족합니다.”
- “현재 풀이에서 막힌 부분은 미분이 아니라 판별식 조건을 세우는 단계입니다.”

        ↓

[15. 이전 학년 개념 결손 진단]
→ 오답 또는 헷갈림이 발생하면 Deepen은 현재 개념/유형뿐 아니라 이전 학년 개념 결손 여부를 진단한다.
→ 이는 Deepen의 핵심 차별 기능 중 하나다.

진단 예시:
현재 문제:
- 수학Ⅱ 미분의 활용
- 곡선 밖의 점에서 접선 구하기

겉으로 보이는 약점:
- 미분의 활용 문제 오답

실제 가능 원인:
- 접선의 의미 부족
- 일차함수 기울기 이해 부족
- 함수값 대입 미숙
- 이차방정식 판별식 활용 부족
- 접선과 중근 조건 연결 실패

→ Deepen은 이 중 어떤 선행 개념이 실제 원인인지 사용자의 이력과 풀이 데이터를 기반으로 추정한다.

        ↓

┌────────────────────────────────────┬────────────────────────────────────┐
│ [15-A. 선행 개념 결손 가능성 높음]   │ [15-B. 선행 개념 결손 가능성 낮음]   │
└──────────────────┬─────────────────┴──────────────────┬─────────────────┘
                   ↓                                    ↓

[15-A. 선행 개념 결손 가능성 높음]
→ 현재 유형 문제를 계속 추천하기보다, 먼저 짧은 리캡을 추천한다.
→ 리캡은 사용자를 이전 학년 전체 단원으로 보내는 것이 아니라, 현재 문제 해결에 필요한 최소 개념만 복구한다.

추천 예시:
- 이차방정식의 판별식 2분 리캡
- 함수값 대입 미니 퀴즈
- 일차함수 기울기 리캡
- 원래 문제 재도전

        ↓

[15-B. 선행 개념 결손 가능성 낮음]
→ 현재 유형 숙련도 부족으로 판단한다.
→ 같은 유형의 쉬운 문제, 유사 문제, 변형 문제를 추천한다.

추천 예시:
- 같은 Pattern의 쉬운 문제 2개
- 유사한 중간 난이도 문제 1개
- 원래 문제 재풀이
- 한 단계 높은 응용 문제

        ↓

[16. 오답노트 / 헷갈린 문제 저장]
→ 문제 결과는 자동으로 저장된다.
→ 오답노트는 단순 문제 저장소가 아니라 원인별 분류 시스템이어야 한다.

분류:
- 개념 오답
- 유형 인식 실패
- 계산 실수
- 조건 해석 오류
- 그래프 해석 오류
- 풀이 시간 초과
- 힌트 의존
- 맞았지만 헷갈림
- 재오답
- 이전 학년 개념 결손

→ 특히 “이전 학년 개념 결손” 태그가 붙으면, 해당 문제는 현재 단원 오답노트뿐 아니라 선행 개념 리캡 목록에도 연결된다.

        ↓

[17. 학습자 상태 업데이트]
→ 사용자의 결과는 개념 단위와 유형 단위, 그리고 선행 개념 단위로 함께 누적된다.

업데이트 대상:
1. Concept State
   - 개념 이해도
   - 리캡 필요 여부
   - 선행 개념 결손 가능성

2. Pattern State
   - 유형 숙련도
   - 정답률
   - 평균 풀이 시간
   - 힌트 의존도
   - 헷갈림 점수

3. Item Attempt State
   - 개별 문제 풀이 기록
   - 오답 여부
   - 풀이 시간
   - AI 질문
   - 풀이과정 업로드

예시:
미분의 활용:
- 곡선 위 접선: 0.82
- 곡선 밖 접선: 0.41
- 증가/감소 판단: 0.74
- 극값 활용: 0.66
- 그래프 개형 추론: 0.38
- 실근 개수 연결: 0.29

이전 학년 개념:
- 이차방정식의 판별식: 0.44
- 일차함수의 기울기: 0.61
- 함수값 대입: 0.57

        ↓

[18. 그래프에 약점 및 결손 개념 반영]
→ 학습자의 약점 유형과 선행 개념 결손이 그래프에 시각적으로 반영된다.
→ 단순히 “미분이 약하다”가 아니라, 더 구체적으로 표시되어야 한다.

예시:
- 미분의 활용 중 곡선 밖 접선 유형 약함
- 해당 유형의 선행 개념인 판별식 결손 가능성 높음
- 접선 조건과 중근 조건 연결이 불안정함

표시 방식:
- 약점 유형 노드 강조
- 반복 오답 유형 경고 표시
- 이전 학년 결손 개념 노드 표시
- 현재 유형과 결손 개념 사이의 연결선 강조
- 추천 복습 경로 표시

        ↓

[19. 다음 학습 액션 추천]
→ 시스템은 사용자의 상태를 기반으로 다음 행동을 추천한다.
→ 추천은 크게 두 갈래로 나뉜다.

1. 선행 개념 결손형 추천
   - 이전 학년 개념 리캡
   - 리캡 퀴즈
   - 기초 적용 문제
   - 원래 문제 재도전

2. 유형 숙련도 부족형 추천
   - 같은 유형 쉬운 문제
   - 같은 유형 중간 문제
   - 유사 문제
   - 응용 유형 문제
   - 킬러 대비 문제

추천 예시:
오늘의 추천 루트:
1. 이차방정식의 판별식 2분 리캡
2. 중근 조건 미니 문제 1개
3. 곡선 밖 접선 쉬운 문제 1개
4. 2024 6모 유사 문제 1개
5. 원래 틀린 문제 재도전

        ↓

[20. 유사 문제 추천]
→ Deepen은 문제를 단순 단원 기준으로 추천하지 않는다.
→ 유형 집합, 유형 조합 방식, 응용 깊이, 선행 개념 연결성을 기반으로 유사 문제를 추천한다.

유사도 기준:
- 유형 집합 유사도
- 유형 조합 방식
- 응용 깊이
- 필요한 선행 개념
- 사용자의 약점 유형 포함 여부

예시:
문제 A:
- 곡선 밖 접선
- 접점 매개변수 설정
- 판별식 활용

문제 B:
- 외부 점 접선
- 접점 변수 설정
- 실근 조건 활용

→ 겉보기 문장은 다르지만 풀이 구조가 유사하므로 추천 가능

        ↓

[21. 문제 생성 에이전트]
→ 사용자가 특정 유형을 더 연습하고 싶으면 AI가 새로운 문제를 생성한다.
→ 단, 초기에는 고난도 킬러 문제 생성보다 기본/중간 난이도 변형 문제 생성부터 시작하는 것이 안전하다.

생성 흐름:
1. 기준 개념/유형 선택
2. 필요한 선행 개념 확인
3. 기준 기출 구조 분석
4. 새 조건 생성
5. 풀이 가능성 검증
6. 정답 검증
7. 해설 생성
8. 사용자에게 제공

예시:
사용자가 “곡선 밖 접선 유형 더 풀고 싶어”라고 하면,
Deepen은 같은 Pattern을 유지하되 숫자, 함수 형태, 조건 결합을 바꾼 새 문제를 생성한다.

        ↓

[22. 반복 학습 루프]
→ 사용자는 추천된 리캡, 유사 문제, 생성 문제, 오답 복구 문제를 다시 푼다.
→ 결과는 다시 그래프와 숙련도 벡터에 반영된다.

최종 반복 루프:
기출 기반 중요 개념·유형 확인
→ 개념/유형 선택
→ 필요한 이전 학년 개념 확인
→ 필요 시 리캡
→ 관련 문제 풀이
→ 정답/오답/헷갈림 분석
→ AI 피드백
→ 선행 개념 결손 진단
→ 유형 숙련도 업데이트
→ 약점/결손 개념 그래프 반영
→ 유사 문제/복습 개념/다음 유형 추천
→ 반복
