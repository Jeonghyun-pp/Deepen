Deepen 서비스 플로우차트 - 노드 역할 및 미구현 상세

기존 플로우차트의 각 노드를 제품/시스템 역할 기준으로 재정리하고, 미구현 또는 상용화 전에 보강해야 할 항목을 명시한 상세판입니다.

구현됨 부분 구현 미구현/보강 필요 외부/기반

1. 진입 & 인증

landing-v2
구현

app/landing-v2, app/page.tsx

역할: 신규 사용자가 서비스 가치를 확인하고 로그인으로 진입하는 마케팅/온보딩 화면.

보강: 요금제, 약관, 개인정보 처리, 제품 데모 CTA는 상용 전환 전에 추가 필요.

/login + Magic Link
구현

app/login/page.tsx

역할: 이메일 OTP 링크를 발송하고 사용자를 인증 플로우로 보냄.

보강: resend 제한, abuse 방지, 커스텀 SMTP, 로그인 이벤트 로깅.

/auth/callback + proxy
구현

app/auth/callback/route.ts, proxy.ts

역할: Supabase code를 세션으로 교환하고 보호된 화면 접근 시 세션을 갱신.

보강: SSO, 계정 삭제, 조직/팀 권한 모델은 아직 없음.

2. PDF 업로드

Upload UI
구현

app/upload, UploadDropzone, UploadPanel

역할: PDF 선택/드롭, 업로드 요청, 최근 문서 상태 표시.

보강: 대용량 업로드 재개, 중복 파일 감지, 사용자별 업로드 quota.

/api/documents/upload
구현

app/api/documents/upload/route.ts

역할: PDF/30MB 검증, Storage 저장, documents row 생성, 202 응답.

보강: 바이러스 스캔, 파일 hash, 멱등 업로드 키, tenant별 제한.

after() 백그라운드 시작
부분

Next after(), processDocument()

역할: 응답 이후 PDF 처리 파이프라인을 fire-and-forget으로 실행.

미구현: 명시적 job queue, retry count, dead-letter, 중복 실행 lock, step별 진행률.

3. PDF 처리 파이프라인

parsePdf
구현

lib/pipeline/parse-pdf.ts

역할: PDF에서 페이지/텍스트 chunk를 추출하고 문서 pageCount를 기록.

보강: 표/수식/이미지 OCR, 깨진 PDF 복구, 페이지별 실패 격리.

groupSections
구현

lib/pipeline/group-sections.ts

역할: chunk를 섹션 단위로 묶어 LLM 추출 단위로 변환.

보강: 목차/헤딩 기반 구조화, 긴 문서 chunk window 최적화.

extractNodesFromSections
구현

lib/pipeline/extract-nodes.ts

역할: LLM으로 개념 노드와 prerequisite/contains 관계를 추출.

보강: schema 검증 강화, hallucination 검출, 섹션별 재시도/부분 실패 UI.

dedup/filter/token_usage
구현

dedup.ts, filter-nodes.ts, token-usage.ts

역할: 중복 노드 병합, 노이즈 제거, LLM 토큰 사용량 기록.

보강: 사용자별 비용 cap, 품질 평가 샘플링, 추출 결과 버전 관리.

4. 그래프 탐색

/api/graph/current
구현

app/api/graph/current/route.ts

역할: 로그인 사용자의 nodes/edges를 읽어 그래프 UI 초기 데이터로 제공.

보강: pagination, incremental sync, 큰 그래프용 서버 필터링.

GraphShell + Reagraph
구현

GraphShell.tsx, GraphCanvas.tsx

역할: 2D/3D 그래프 렌더링, 선택, hover preview, layout 변경 제공.

보강: 수천 노드 이상 성능 최적화, 클러스터링, progressive loading.

Filter/Search/Gaps
구현

useGraphData, graphSelectors

역할: 노드 타입 필터, 로컬 탐색, gap node 계산, 클라이언트 검색.

보강: 서버 검색, vector search, relevance ranking, 저장 가능한 필터 preset.

NodeSources
구현

/api/nodes/{id}/sources

역할: 특정 노드가 어떤 문서 chunk에서 왔는지 출처를 보여줌.

보강: PDF 원문 페이지 jump, chunk highlight, citation confidence.

5. 화이트보드

/graph/whiteboard
부분

WhiteboardShell, WhiteboardCanvas

역할: xyflow 기반으로 노드를 배치하고 그룹/관계를 시각적으로 편집.

미구현: 직접 진입 시 실제 DB가 아닌 sample data 초기화 가능성.

whiteboardStore
부분

app/graph/_store/whiteboardStore.ts

역할: 노드 위치, 확장 상태, section grouping을 localStorage에 저장.

미구현: 위치/그룹/레이아웃의 서버 영속화 및 다른 기기 동기화.

노드/엣지 편집
부분

/api/nodes, /api/edges

역할: 일부 수동 생성/삭제를 DB에 반영.

미구현: 모든 편집 액션의 undo/redo, optimistic rollback, audit trail.

6. 문서 관리

/documents 목록
구현

app/documents/page.tsx

역할: 업로드한 PDF 목록, 처리 상태, 페이지 수, 실패 메시지를 표시.

보강: 검색/정렬/태그, 문서별 사용량, 재처리 버튼.

문서 삭제
구현

역할: Storage PDF 삭제, document/chunk/mapping cascade, orphan node 정리.

보강: soft delete, 복구, storage 삭제 실패 보상 job.

문서 상세
부분

GET /api/documents/{id}

역할: 문서 row 자체 조회는 가능.

미구현: 문서별 전체 chunks/sections/nodes/edges를 탐색하는 상세 화면.

8. Agent Chat

SSE Chat API
구현

/api/agent/chat, useAgent

역할: 사용자 질문을 받아 LLM 응답과 tool event를 스트리밍.

보강: session persistence, conversation history DB 저장, 중단/재개.

Tool Calling
구현

역할: 그래프 질의, 경로 찾기, 개념 추출, OpenAlex 논문 검색을 수행.

보강: tool별 timeout, rate limit, result cache, 실패 fallback.

Approval Flow
부분

approval.ts, /api/agent/approve

역할: add_node/add_edge 같은 mutation tool은 사용자 승인 후 실행.

미구현: in-memory Map이라 서버리스/멀티 인스턴스에서 불안정. DB/Redis queue 필요.

상용 비용/권한 제어
미구현

planned

역할: LLM/API 비용과 위험한 도구 실행을 사용자/요금제별로 제한.

미구현: quota, spend cap, audit log, abuse detection, plan-based permissions.
