## 배경

현재 그래프는 38개 노드(paper 9 / concept 8 / technique 6 / application 5 / question 5 / memo 3 / document 2)로 규칙 검증용으론 부족. 타입별 색상·크기 차등(`NODE_BASE_SIZE`), 엣지 3단계(citation/contains/기타), degree 보정은 이미 들어가 있음. force 파라미터는 `GraphCanvas.tsx`에 하드코딩 (`centerInertia: 0.8 / nodeStrength: -350 / linkDistance: 170`).

## 시각화 규칙 요소 — 4개로 부족

`크기 / 거리 / 엣지 / 색상` 4개는 정적 구조만 표현. 탐색 도구로 동작하려면 최소 3개 추가 필요:

1. **라벨 표시 규칙** — 상위 N + 줌 기반 단계적 노출 + hover 확장
2. **투명도 / 강조 상태** — focus(선택) / context(1-hop) / dimmed(나머지) 3단계
3. **모양 / 아이콘** — 색맹 대응 + 7개 타입 색만으로 구별 약함

부차적: 클러스터 hull, 줌 LOD, 애니메이션 전환.

## 벤치마킹 핵심 (상세는 대화 히스토리 참조)

- **Obsidian** — hover dim + depth 슬라이더 + text fade threshold (미니멀, 사용자 위임)
- **Research Rabbit** — 엣지 타입별 뷰 탭 분리, 상태 기반 색상
- **Connected Papers** — 크기=중요도 / 색 진하기=시간축 이중 인코딩
- **Kumu** — 엣지 타입을 색·굵기·점선·화살표 4채널로 분리
- **Neo4j Bloom** — 타입별 아이콘+색상 이중 인코딩, caption 필드 타입별 지정
- **Cosmograph** — "상위 N 라벨만" 원칙

이론 근거: Munzner의 visual channel ranking — 위치 > 길이 > 각도 > 색상 > 모양.

## Obsidian vs Deepen — 핵심 차이

| | Obsidian | Deepen |
|---|---|---|
| 노드 타입 | 1개 | 7개 |
| 엣지 타입 | 1개 | 11개 |
| 의미 부여 | 사용자 위임 (Groups) | 스키마 내장 |
| 그래프 역할 | 보조 뷰 | 1st-class 캔버스 탭 |
| 컨트롤 노출 | 4탭 floating 패널 | 하드코딩 |

→ Obsidian은 "스타일 사용자 위임"이 가능하지만, Deepen은 **디자이너가 default를 제대로 설계해야** 함 (타입이 의미 전달의 1차 채널이라).
