## 1. Executive Summary

오르조는 입시(수능·공무원·자격증) 태블릿 문제풀이 앱으로, 4년 CAGR 180% / 누적 100만 다운로드 / 2024년 신규 40만 / 매출 5배 성장을 기록한 한국 에듀테크의 최상위 사례다 [13][18]. 같은 입시 도메인·같은 Anthropic Claude 멀티모달 스택·같은 학원 SaaS 진출 경로를 갖는 Deepen 입장에서, **오르조는 회피 대상이 아니라 가장 잘 검증된 레퍼런스 제품**이다.

이번 분석에서 발견한 4가지 핵심 finding:

1. **AI 코치는 풀이 컨텍스트 내부에서만 호출된다** — 별도 챗봇 탭으로 빠지지 않음. "추천 질문 5칩 + 자유 입력" 하이브리드. **빈칸 문제로 변형해주세요** 칩이 비범한 디자인 (수동 학습 → 능동 출제 전환).
2. **개념카드는 답변 안에 끼워넣는 인서트** 형태로 구현되며, **DAG·prerequisite 시각화는 부재** — Deepen 3계층 그래프와 역추적은 명확한 차별 우위로 남는다.
3. **펜슬 UX는 PencilKit 기반 + 1초 채점 + 문항별 타이머 + 자동·수동 이원 오답노트 + 듀얼 패널 + 지문 고정 스크롤**의 표준 셋이 정립되어 있다. Deepen Q2 OCR 베타가 그대로 모방하면 되는 영역.
4. **오르조는 듀오링고형 스트릭/푸시 게이미피케이션이 거의 노출되지 않는다** — 대신 (a) AI 일 50회 캡 = 일 리셋 hook, (b) 챌린지 퀘스트(기존 문제 재조합), (c) "월 +10시간" 통계 카피, (d) 오르조 클래스의 교사 감시(사회적 accountability)로 retention을 만든다. Deepen은 입시 도메인에 자연스러운 retention 노선을 의식적으로 결정해야 한다.

| 측면 | 즉시 차용 | 차별화 유지 |
|---|---|---|
| A · AI 코치 호출 UX | 풀이 컨텍스트 호출 / 5칩 추천 / 답변 패널 / 일 사용량 카운터 | 단계별 펼치기에 DAG 인용 결합 |
| B · 개념카드 | AI 응답 인서트형 미니 카드 / 자동 개념 식별 | DAG/prerequisite 그래프 시각화 |
| C · 펜슬 입력 | PencilKit / 1초 채점·문항타이머 / 자동·수동 이원 오답노트 / 지문 고정 스크롤 | PDF 드래그→AI hero 인터랙션 |
| H · Retention | 챌린지 퀘스트 / "월 +X시간" 카피 / AI 캡 일 리셋 | 사회적 accountability hook (메이트/보호자 리포트) |

## 2. Deepen ↔ 오르조 자산 매핑

| Deepen 자산 (D2SF deck) | 오르조 대응 자산 | 격차 |
|---|---|---|
| 3계층 그래프 (Concept-Pattern-Item) | SDKT (자체 딥러닝 지식 추적) — 단원 단위로 추정 | Deepen 우위 (유형 단위) |
| Prerequisite DAG 역추적 | 부재 — 단일 개념카드만 | **Deepen 명확한 차별** |
| 1~3분 리캡카드 | 개념카드 (응답 인서트) | Deepen이 **DAG 기반 카드 시퀀싱**으로 차별화 가능 |
| 8가지 오답 분석 + 정답이지만 헷갈림 | 자동·수동 이원 오답노트 (정오답 이분법) | Deepen 우위 (헷갈림 별도 분류) |
| AI 학습 코치 (코치 AI) | AI 코치 2.0 + Max | 호출 UX·추천 질문 칩은 오르조가 1.5년 앞서 있음 — 차용 대상 |
| 풀이과정 OCR (Q2 베타) | PencilKit + 자동채점 표준 셋 | 오르조가 4년 운영 — 통째 모방 |
| 학원 SaaS (Q4 베타) | 오르조 클래스 (공교육 30곳+) | 오르조가 운영 모델 검증 완료 — 패키지 차용 |

핵심 통찰: **Deepen의 차별은 "유형×결손 그래프"라는 분석 엔진**에 있고, **오르조의 강점은 "풀이 환경 자체의 완성도"**에 있다. 두 자산은 충돌하지 않는다 — 오르조의 풀이 환경 위에 Deepen의 분석 엔진을 얹는 그림이 가능하다.

## 3. 영역 A — AI 코치 호출 UX

### A.1 트리거 위치 (★★★)
오르조 AI 코치는 **문제풀이 화면 하단 'AI 코치' 버튼**으로만 호출된다 [1]. iPad 앱 4.34.0+ 에서 활성화. 즉 **별도 챗 탭으로 진입할 수 없고 풀이 컨텍스트 안에서만 작동**한다는 의식적 설계다.

**왜 중요한가**: NotebookLM·범용 LLM은 "대화 탭"에서 시작해 대화가 메인이 된다. 오르조는 정반대로 **풀이가 메인, AI는 도구**라는 위계를 UX에 박아 넣었다. Deepen pitch deck의 "5분짜리 강사 설명을 모든 학생에게 동일 비용으로"라는 핵심 가치 제안과 정확히 일치 — Deepen도 풀이/문제 화면이 hero가 되고, 코치는 in-line invoke되는 구조여야 한다.

**Deepen 적용**:
- 그래프 화면, PDF 강의안 화면, 풀이 화면 어디에서도 **챗 탭으로 이동시키지 말 것**.
- 코치 호출은 항상 현재 컨텍스트 위에 띄우는 패널 또는 바텀시트.
- 풀이 화면 기준 하단 우측 또는 우측 사이드 가장자리 sticky 버튼이 표준 위치.

### A.2 추천 질문 5칩 (★★★)
오르조는 다음 5개 추천 질문을 칩 형태로 노출한다 [1]:

1. "힌트를 제공해주세요"
2. "~~~의 뜻을 알려주세요"
3. "2번은 왜 오답인가요?"
4. "더 자세히 설명해주세요"
5. "이 문제를 빈칸 문제로 변형해주세요"

5번이 특히 비범하다. 학생이 **수동 학습자에서 능동 출제자로 전환**되는 칩이고, 이 칩 하나가 같은 문제를 다른 형태로 3~5번 더 풀게 만든다 — 사실상 retention 엔진의 일부다.

**Deepen 적용 (수학 컨텍스트로 번역)**:

| 오르조 칩 | Deepen 수학 칩 (제안) |
|---|---|
| 힌트 제공 | "첫 한 줄 힌트만 주세요" |
| 단어 뜻 | "이 문제에 쓰인 ○○ 정의 알려주세요" |
| 오답 근거 | "왜 X 풀이는 안 되나요?" |
| 더 자세히 | "이 단계 더 풀어서 보여주세요" |
| 빈칸 변형 | **"같은 유형 다른 숫자로 다시 풀고 싶어요"** ← 핵심 |

다섯 번째 칩은 Deepen 3계층 그래프의 **Pattern 노드와 직결**된다 — 학생이 누른 순간 같은 Pattern의 다른 Item을 추천하는 자연스러운 진입점이 된다. **즉 오르조의 단순 "변형 출제"를 Deepen은 "유형 숙련도 +0.05" 같은 측정 가능한 학습 액션으로 격상**할 수 있다.

### A.3 응답 흐름 — 단계 펼치기 + 시각 앵커 (★★★)
- **수학**: 2024-12 출시한 대화형 수학 코치는 "필요한 개념·공식 자동 식별 → 단계별 설명 → 지속 대화로 맞춤 풀이" [4][5]. 즉 **한 번에 전체 풀이를 토하지 않고 단계 펼치기 UX**.
- **국어/영어**: AI가 답변 시 **지문 본문에 직접 하이라이트** [1]. 답변에 시각 앵커가 남는다는 뜻이다.

**Deepen 적용 (PDF-centric 피벗과 결합)**:
- 학생이 PDF 강의안에서 드래그 → AI 코치 호출 → 응답이 텍스트만 던지는 게 아니라 **드래그 영역과 답변에서 언급한 개념을 그래프 노드와 시각 연결**.
- 펜슬로 풀이를 적은 영역(OCR 후 좌표 보존)을 **AI가 답변 시 해당 영역을 다시 하이라이트**하며 "여기서 막혔습니다"라고 가리키는 것 — 이게 pitch deck의 "어디서 막혔는지" 메시지를 시각 UX로 구현하는 가장 강력한 방법이다.

### A.4 답변 패널의 인터랙션 디테일 (★★★)
오르조 패널에는 다음이 들어 있다 [1]:
- 답변 메시지 옆 **화살표 아이콘 → 그 답변 특정 내용에 추가 질문**
- 패널 **크기 조절** (펜슬로 풀이하면서 좌측에 채팅을 두는 듀얼 사용 시나리오 의식)
- 우상단 **대화 초기화** 버튼

**Deepen 적용**:
- 답변 메시지마다 "이 부분 더" 화살표 — 단일 응답이 아니라 **드릴다운 트리** UX.
- 패널 크기 조절은 펜슬 풀이 환경에서 필수.
- 대화 초기화 = 새 문제로 넘어갈 때 컨텍스트 리셋. 이게 없으면 이전 문제 풀이가 다음 문제 답변을 오염시킨다.

### A.5 사용량 캡이 retention 엔진 역할을 한다 (★★★)
- 무료/프리미엄: **총 5회만 체험**
- 프리미엄 플러스: **매일 50회**
- Max: 무제한

이 구조의 핵심은 "매일 50회"의 **일 리셋**이다 — 듀오링고 하트와 동일한 기제. 학생은 캡이 차면 다음 날 리셋을 기다리며 자연스럽게 매일 앱을 연다. 사용량 표시가 명시적 카운터로 노출됐을 가능성이 매우 높다 (★★ 추정).

**Deepen 적용**: Q3 유료 런칭의 Pro/Pro+ 가격 티어 설계 시, **사용량 게이트가 retention 엔진**임을 의식할 것. "Pro 일 30회 / Pro+ 무제한" 같은 구조면 Pro 사용자가 매일 돌아온다. 무제한만 있으면 이 hook이 사라진다.

## 4. 영역 B — 개념카드 (Concept Card)

### B.1 형태: 답변 안에 끼워넣는 인서트 (★★★)
오르조 공식 표현: "기존에는 풀이 및 해설에만 집중했지만, 이제는 '개념카드'를 활용하여 개념까지 상세하게 설명" [3]. 즉 **별도 위키 페이지·별도 탭이 아니라 AI 응답 흐름 안에서 호출되는 카드 컴포넌트**.

**Deepen 적용 (가장 직접적 차용 영역)**:
Deepen pitch deck의 "1~3분 리캡카드"는 사실상 같은 컴포넌트다. 차이는 호출 trigger:
- 오르조: 학생 질문에 대한 답변 안에 자동 삽입
- Deepen: 학생 오답 → prerequisite 결손 자동 도출 → 리캡카드 자동 띄움

**즉시 차용할 디자인 요소**:
1. 카드는 풀이 화면 위에 띄우는 모달/바텀시트 (전체화면 X — 풀이 컨텍스트 유지)
2. 카드 헤더에 학년/단원 배지 (오르조: 단원명 / Deepen: "중3 · 이차방정식 판별식" 같은 학년 표기 — pitch deck 9페이지 리캡카드 디자인과 일치)
3. "왜 필요한가" 한 줄 + "3줄 핵심" + "확인 퀴즈" + "원래 문제로 돌아가기" 버튼 (pitch deck 9페이지 그대로 + 오르조 표현 차용)

### B.2 자동 개념 식별 (★★★)
"AI 코치가 필요한 개념과 공식을 자동으로 식별 → 단계별 설명" [4][5]. 즉 학생이 "이 개념 알려줘"라고 묻지 않아도 **풀이 단계에서 부족한 개념을 시스템이 추론·삽입**한다.

**Deepen 적용**:
이 부분이 Deepen 3계층 그래프와 정확히 맞물린다. 오르조는 "AI가 추론해서 카드 띄움"인데 Deepen은 **"학습자 그래프에서 약점 노드를 데이터로 자동 검출 → prerequisite DAG 역추적 → 카드 시퀀스 자동 구성"**까지 갈 수 있다. 같은 UX 컴포넌트지만 백엔드가 한 단계 더 깊다.

### B.3 결정적 부재: DAG/prerequisite 시각화 (★★★ 부재 자체가 finding)
공식 자료, 보도자료, 후기 어디에도 **개념 간 연결 그래프** 시각화는 등장하지 않는다. 오르조의 개념카드는 "문제 옆에 붙는 단일 카드"에 머문다.

**Deepen 차별 영역으로 유지**:
- pitch deck 8페이지 "개인 학습 상태 지도"는 오르조에 없는 자산.
- pitch deck 9페이지 "선행 결손 후보 → 1~3분 리캡 → 원래 문제 복귀" 4단 사이클도 오르조에 없음.
- 즉 Deepen 차별 메시지: **"오르조 개념카드는 한 장, Deepen 리캡카드는 그래프 위에서 자동 시퀀싱되는 카드"**.

## 5. 영역 C — 태블릿 + 스마트펜슬 입력 UX

### C.1 PencilKit 기반 + 커스텀 도구 (★★★)
"애플 PencilKit을 활용해서 개발 / 원하는 색상과 두께로 커스텀 필기도구" [6][7]. 자체 ink 엔진을 만들지 않는다는 정책.

**Deepen 적용**: Q2 OCR 풀이과정 베타는 **PencilKit 위에서 시작**할 것. ink 엔진 자체 개발은 자원 낭비. 단 OCR/스트로크 보존을 위해 PencilKit drawing → custom export 파이프라인은 직접 구현 필요.

### C.2 1초 자동채점 + 문항별 타이머 (★★★)
"1초 만에 채점 / 내가 쓴 답, 문제별 정답 및 풀이 시간 확인" [6][7]. 자동채점 결과 페이지에 **문항별 풀이 시간이 함께** 노출된다.

이 디자인의 함의가 크다 — 학생이 "정답을 맞췄는가"만이 아니라 "정확하게 풀면서 시간 줄이는 연습"을 하게 된다. 사용자 후기: "어디서 많이 틀리고, 뭐 부족하고, 어디에 시간 많이 걸리는지 직관적으로 볼 수 있어서 셀프 피드백에 진짜 도움됐다" [7].

**Deepen 적용**:
- 채점 결과 화면에 **시간 분포** 무조건 포함 — pitch deck 10페이지 "채점은 3분기다"의 "정답이지만 헷갈림" 카테고리는 **풀이 시간 + 자신감 + 힌트 사용 횟수**로 측정해야 함. 오르조가 시간 측정 패턴을 검증해줬음.
- 약점 시각화는 (a) 유형별 정답률 (b) 유형별 풀이시간 (c) 자신감 셀프 리포트 — 3축 동시.

### C.3 오답노트의 이원화: 자동 + 수동 (★★★)
"원하는 문제들에 태그를 달아 따로 모아보기 / 2번 이상 틀린 문제만 모아보는 문항표" [6]. 두 입력 경로:
- **사용자 능동**: 태그
- **시스템 자동**: 2회+ 오답 자동 누적

**Deepen 적용**:
이 이원화는 그대로 모방 + Deepen 자산 결합:
- 사용자 능동: 학생이 즐겨찾기/별표
- 시스템 자동: **2회+ 오답 + "정답이지만 헷갈림"** 두 경로 모두 자동 누적
- 시스템 자동 우선 노출 (오답노트 = 시스템이 만들어주는 게 default), 사용자 태그는 보조 필터

### C.4 듀얼 모드 + 지문 고정 스크롤 (★★★)
- "문제집과 해설지를 화면 이동 없이 동시 표시" [9]
- "긴 지문이 문제를 가릴 때 문제 쪽은 고정한 채 지문만 스크롤" [11]

후자 디테일이 매우 구체적인 friction 제거다. 국어 비문학 긴 지문에서 학생이 지문↔문제 왔다갔다 하는 동선을 한 화면에 묶어버린다.

**Deepen 적용 (PDF-centric 피벗과 정확히 일치)**:
- PDF 강의안 + 그래프 + AI 코치를 같은 화면에 동시 노출하는 **3-pane 레이아웃**의 정합성을 오르조가 검증해줬다.
- 한 패널이 길면 그 패널만 스크롤되고 다른 패널은 고정 — UX 복잡해 보이지만 학생이 실제로 적응한다는 신호.

### C.5 PDF 학습 모드 (★★)
오르조 사용 가이드 PDF [15]에서 PDF 직접 업로드 + AI 코치 동작 가능성 시사. 본문 미접근으로 정확한 기능은 미확인이지만, **오르조도 PDF 입력 경로를 이미 열어두고 있을 가능성**이 있다 → Deepen PDF-centric 피벗이 절대적 차별이라고 가정하면 안 됨. PDF 위에서 "드래그 → AI" 인터랙션의 매끄러움 자체가 차별 포인트가 되어야 한다.

## 6. 영역 H — Retention 디자인

### H.1 핵심 finding: 듀오링고형 게이미피케이션이 거의 없다 (★★★ 부재 자체가 finding)
6회 이상 검색에도 **streak·연속출석·뱃지·푸시 리마인더**에 대한 구체적 기능 설명이 공식·블로그·후기 어디에도 없다. 이는 우연이 아니라 의식적 결정으로 보인다 — 입시 도메인은 학생이 이미 외재적 압력(시험)을 받고 있어서, 추가 게이미피케이션이 오히려 "장난스러운 앱"으로 저평가될 위험이 있다.

**Deepen 의사결정 포인트**:
- 입시 컨텍스트에서 streak/하트 도입은 **신중**해야 한다.
- 대신 오르조가 쓴 retention 메커니즘 4가지를 의식적으로 차용/재해석할 것 (H.2~H.5).

### H.2 챌린지 퀘스트 — 기존 콘텐츠 재조합 (★★)
브런치 비교 글: "추가적인 문제 제작 없이 기존 문제를 조합한 챌린지 퀘스트" [11]. 즉 새 콘텐츠 생산 없이 **데일리/위클리 미션 단위가 retention hook 역할**.

**Deepen 적용**:
Q2 시점부터 가능한 가장 저비용 retention hook:
- 매일 아침 "오늘의 약점 유형 3문제" — 학습자 그래프에서 자동 추출
- 주간 "이번 주 막힌 prerequisite 복습" — DAG 역추적 결과를 묶어 1주 1세션
- **콘텐츠 추가 없이** 기존 기출 + 유형 매칭만으로 데일리 미션 생성 가능 (Deepen은 이미 Item 레이어를 갖고 있으니)

### H.3 "월 +10시간" 통계 카피 (★★★)
복수 출처 반복 인용 — "월 평균 학습 시간이 약 10시간 늘어났습니다" [6][12][13]. 마케팅 카피일 뿐 아니라 **앱 안에서 학생이 자기 통계 화면을 볼 때 동일 framing이 들어가 있을 가능성이 매우 높다**.

**Deepen 적용**:
- 학생 통계 화면에 절대 수치(공부 시간)만이 아니라 **"오르조 사용 전 대비 +X시간"** 같은 **개선 framing**을 노출.
- Deepen 특화: "지난 주 막혔던 유형 N개 → 이번 주 N-2개"같이 **약점 노드 감소**를 retention 카피로.
- 이런 framing은 학생을 **"내가 오늘 공부했나"가 아니라 "내가 발전하고 있나"**에 묶는다.

### H.4 "필수 앱" 의존성 자체가 retention (★★★)
오르조 CEO 안강민 인터뷰 키워드: **"학습 필수 앱"** [13]. 게이미피케이션이 아니라 **"이 앱 없이는 풀이가 불편"**한 의존성을 retention 동력으로 둠.

**Deepen 적용**:
오르조에 비유하면 "자동채점·필기·해설지가 없으면 종이 문제집이 너무 불편" → Deepen은 "유형 단위 약점 추적·prerequisite 역추적·1~3분 리캡이 없으면 풀이가 너무 비효율"이라는 의존성을 만들어야 한다. 즉 **차별 자산 그 자체가 retention** — 부가 hook 없이도 학생이 돌아온다.

### H.5 사회적 accountability — 오르조 클래스의 의외의 retention 메커니즘 (★★★)
교사 인용: "학생도 내 필기나 공부 기록을 선생님이 보고있다는 것을 알기 때문에 태도가 바뀌는 것 같습니다" [14]. **사회적 감시(social accountability)가 알고리즘 푸시보다 강한 hook**이라는 신호.

**Deepen 적용 (B2C에서 응용)**:
B2B 학원 SaaS 진출 전, B2C에서도 사회적 hook을 만들 수 있다:
- **보호자 리포트** (주간 약점 그래프 변화 자동 발송)
- **학습 메이트** (친구 한 명 초대 → 둘의 그래프 비교 / 같은 결손 묶어서 함께 풀기)
- 두 메커니즘 다 듀오링고 streak보다 입시 컨텍스트에 자연스럽다.

### H.6 AI 캡 일 리셋 (★★, A.5와 연계)
A.5에서 다뤘듯, 일별 사용량 캡 자체가 retention 엔진. 별도 streak 없이도 매일 앱을 연다.

- ✅ **A.1 트리거 위치**: 풀이 화면 sticky 코치 버튼 (챗 탭 X) — 처음부터 박을 것

- ✅ **B.1 카드 형태**: 답변 안 인서트 형태로. 별도 위키 페이지 만들 유혹 회피
- ✅ **A.4 답변 패널 인터랙션**: 화살표 추가질문 + 패널 크기 조절 + 대화 초기화

### Q2 (선행 결손 + 리캡 + 풀이 OCR 베타)
- ✅ **C.1 PencilKit**: ink 엔진 자체 개발 X. PencilKit 표준 위에 export 파이프라인만
- ✅ **C.2 1초 채점 + 문항 타이머**: 채점 결과 화면에 시간 분포 무조건 포함
- ✅ **C.3 오답노트 이원화**: 시스템 자동(2회+ 오답 + "헷갈림") + 사용자 태그
- ✅ **C.4 듀얼 패널 + 지문 고정 스크롤**: PDF + 그래프 + 코치 3-pane 레이아웃 검증
- ✅ **B.2 자동 개념 식별**: 학생 오답 → 약점 노드 검출 → prerequisite DAG 역추적 → 카드 시퀀스 자동 구성 (오르조 단일 카드 → Deepen 시퀀스 카드)

### Q3 (유료 런칭, 수능 D-180)
- ✅ **A.5 + F · 가격 티어 + 사용량 캡**: 무료 5회 평생 / Pro 일 N회 / Pro+ 무제한. 일 리셋 hook 의식적 설계
- ✅ **H.3 "월 +X시간" framing**: 학생 통계 화면에 개선 카피 노출 + Deepen 특화 "약점 노드 감소" 카피
- ✅ **H.5 사회적 hook (B2C)**: 보호자 리포트 베타 (학습 메이트는 후속)

### Q4 (학원 SaaS 베타 + ARR 5억)
- ✅ **D · 오르조 클래스 패턴**: 학생용 앱에 "교사가 보고 있다" 메시지 명시적 노출 — 사회적 accountability를 패키지 셀링 포인트로
- ✅ **G · 데이터 자산화 메시징**: Phase 3 IR 카피 프리뷰 ("유형×결손 누적 + Claude 멀티모달")

## 9. 결정해야 할 4가지 (이 보고서를 의사결정 다큐먼트로 활용)

1. **추천 질문 5칩 카피 확정** — 위 A.2 매핑 표가 시안. MVP 첫 빌드 들어가기 전 결정 필요.
2. **개념카드 = 1~3분 리캡카드 통합 명명** — pitch deck "1~3분 리캡카드"와 오르조 "개념카드" 사이에서 Deepen 명칭 확정. 권장: **"리캡카드"** (오르조와 차별화 + Deepen 고유 자산 강조).
3. **Streak 도입 여부** — H.1의 부재 finding을 받아들일지. 권장: **streak/푸시 도입 X**, 대신 H.5 사회적 hook + H.3 개선 framing.
4. **시장 확장 1차 타깃 재검토** — pitch deck Phase 3는 CFA·의사국시인데, 오르조는 공무원→한능검·LEET→글로벌 순으로 갔다. **공무원 시장이 Deepen Phase 3보다 앞에 들어갈지** 의식적 재검토 필요.
